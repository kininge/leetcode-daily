## 💡 Intuition

The problem asks us to find the **greatest common divisor (GCD)** of the **smallest** and **largest** numbers in the array.

At first, you might think you need to compute the GCD of every element in the array.

However, the problem explicitly asks for only two numbers:

* Smallest element
* Largest element

So the task naturally breaks into two steps:

1. Find the minimum and maximum elements.
2. Compute their GCD using the Euclidean Algorithm.

# 🧠 Attempt 1 — Array Traversal + Euclidean Algorithm

## Idea

Traverse the array once.

Maintain

* minimum value
* maximum value

Then compute their GCD.

### Your Solution

```ts
function findGCD(nums: number[]): number {
    let minNum: number = 1000;
    let maxNum: number = 1;

    for (const num of nums) {
        minNum = Math.min(minNum, num);
        maxNum = Math.max(maxNum, num);
    }

    function GCD(a: number, b: number): number {
        if (b === 0) return a;
        return GCD(b, a % b);
    }

    return GCD(maxNum, minNum);
}
```

---

## 🧠 Euclidean Algorithm

The Euclidean Algorithm is based on the identity

```text
GCD(a, b)

=

GCD(b, a % b)
```

### Why does this work?

Suppose

```text
a = 48

b = 18
```

We can write

```text
48 = 18 × 2 + 12
```

Notice

```text
12 = 48 - (18 × 2)
```

Any number that divides both

```text
48
```

and

```text
18
```

must also divide

```text
12
```

Therefore,

```text
GCD(48,18)

=

GCD(18,12)
```

Repeat the process.

```text
GCD(18,12)

↓

GCD(12,6)

↓

GCD(6,0)

↓

6
```

Once the remainder becomes zero,

the current divisor is the greatest common divisor.

## ✅ Solution

```ts
function findGCD(nums: number[]): number {
    let minNum: number = 1000;
    let maxNum: number = 1;

    for (const num of nums) {
        minNum = Math.min(minNum, num);
        maxNum = Math.max(maxNum, num);
    }

    function GCD(a: number, b: number): number {
        if (b === 0) return a;
        return GCD(b, a % b);
    }

    return GCD(maxNum, minNum);
}
```

---

## ⚖️ Why it Works

The algorithm first identifies

* the smallest element
* the largest element

Since the problem only asks for

```text
GCD(minimum, maximum)
```

all other elements can be ignored.

The Euclidean Algorithm correctly computes the greatest common divisor because

```text
GCD(a,b)

=

GCD(b,a%b)
```

This repeatedly reduces the problem size until the remainder becomes zero, at which point the current divisor is the answer.

## ⏱ Complexity

Finding minimum and maximum

```text
O(n)
```

Euclidean Algorithm

```text
O(log(min(minNum, maxNum)))
```

Overall

```text
Time  : O(n)

Space : O(1)
```

## 🏆 Key Takeaways

* Always read the problem carefully—sometimes it asks for the GCD of only specific values, not the entire array.
* The Euclidean Algorithm is the standard way to compute the GCD of two numbers.
* The key identity to remember is

```text
GCD(a,b)

=

GCD(b,a%b)
```

* Finding both the minimum and maximum can be done in a single traversal.
