# 🧩 Problem #2161 – Partition Array According to Given Pivot

**Difficulty:** Medium
**Topics:** `Array`        
**Link:** [Partition Array According to Given Pivot](https://leetcode.com/problems/partition-array-according-to-given-pivot/description)

---

## 📖 Problem Summary

Rearrange the array such that:

```text
All elements < pivot
        ↓
All elements = pivot
        ↓
All elements > pivot
```

Additionally:

```text
Relative ordering must be preserved.
```

Meaning:

If:

```text
nums = [9,12,5,10,14,3,10]
```

Then:

```text
9 appears before 5
```

originally, so:

```text
9 must still appear before 5
```

after rearrangement.

---

## 💡 Intuition

At first glance, this looks like a classic partition problem.

Many people immediately think:

```text
Quick Sort Partition
```

using:

```text
Two Pointers
```

However:

```text
Quick Sort Partition
❌ does not preserve order
```

This problem requires:

```text
Stable Partition
```

which means:

```text
Relative order must remain unchanged.
```

---

## 🎯 Key Observation

Every number belongs to exactly one of three groups:

```text
< pivot
= pivot
> pivot
```

If we can collect these groups while preserving order, we can simply place them together.

---

## 🚀 Evolution of Thought Process

```text
Need partition around pivot
        ↓
Need relative ordering
        ↓
Regular partition won't work
        ↓
Group elements into categories
        ↓
< pivot

        ↓
Rebuild final array
```

## ✅ Why It Works

We process elements in original order.

Therefore:

```text
Small elements preserve order
Large elements preserve order
```

And pivot values naturally stay in the middle.

---

## 💻 Code

```ts
var pivotArray = function(nums, pivot) {
    const ans = [];

    let pivotNumCount = 0;

    for(const num of nums){
        if(num < pivot) ans.push(num);
        if(num === pivot) pivotNumCount++;
    }

    for(let i = 0; i < pivotNumCount; i++){
        ans.push(pivot);
    }

    for(const num of nums){
        if(num > pivot) ans.push(num);
    }

    return ans;
};
```

## ✅ Why It Works

Both helper arrays preserve insertion order.

Therefore:

```text
9 remains before 5
5 remains before 3

12 remains before 14
```

which satisfies the stable ordering requirement.

---

## 💻 Code

```ts
function pivotArray(nums: number[], pivot: number): number[] {
    const n = nums.length;

    const smallNums = [];
    const largeNums = [];

    for(const num of nums){
        if(num < pivot) smallNums.push(num);
        else if(num > pivot) largeNums.push(num);
    }

    const equal = n - smallNums.length - largeNums.length;
    const lessThanEqual = smallNums.length + equal;

    for(let i = 0; i < n; i++){
        if(i < smallNums.length){
            nums[i] = smallNums[i];
        }
        else if(i < lessThanEqual){
            nums[i] = pivot;
        }
        else{
            nums[i] = largeNums[i - lessThanEqual];
        }
    }

    return nums;
}
```

## 📝 Key Takeaways

* The phrase:

```text
Maintain Relative Order
```

is the biggest clue in the problem.

* Stable partition is different from classic quicksort partition.

* Grouping elements into categories often simplifies array rearrangement problems.

* Sometimes the cleanest solution is simply:

```text
Collect
→ Organize
→ Rebuild
```

---

## ⚠️ Common Mistakes

### Using Quick Sort Partition

Example:

```text
[9,12,5,10,14,3,10]
```

may become:

```text
[3,5,9,10,10,14,12]
```

which violates relative ordering.

---

### Forgetting Pivot Elements

Many solutions correctly collect:

```text
< pivot
> pivot
```

but forget to place:

```text
= pivot
```

in the middle.

---

### Breaking Stable Ordering

Sorting the smaller or larger groups separately is incorrect.

The original ordering must be preserved.
