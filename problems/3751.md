# 🧩 Problem #3751 – Total Waviness of Numbers in Range I

**Difficulty:** Medium     
**Topics:** `Simulation`    
**Link:** [Total Waviness of Numbers in Range I](https://leetcode.com/problems/total-waviness-of-numbers-in-range-i/description)

---

## 📖 Problem Summary

For every number in the range `[num1, num2]`, calculate its **waviness**.

A digit is:

* **Peak** → greater than both neighboring digits
* **Valley** → smaller than both neighboring digits

Rules:

* First and last digits can never contribute.
* Numbers with fewer than 3 digits have waviness `0`.

Return the total waviness of all numbers in the range.

---

## 💡 Intuition

At first glance, this looks like a classic:

```text
Range Query
+
Digit Property
```

which often leads to **Digit DP**.

However, the constraints are small:

```text
num2 ≤ 100000
```

Maximum numbers to process:

```text
100000
```

Maximum digits:

```text
6
```

So even a direct simulation is completely feasible.

Instead of overcomplicating the solution, we can simply:

1. Visit every number in the range.
2. Count its peaks and valleys.
3. Add the result to the answer.

---

## 🎯 Key Observation

A digit only depends on:

```text
Previous Digit
Current Digit
Next Digit
```

Therefore, checking waviness for one number requires only:

```text
O(number of digits)
```

Since:

```text
Maximum digits = 6
```

the total work remains small.

## ⚡ Approach

### Step 1: Convert Number → Digit Array

Example:

```text
120
```

Stored as:

```text
[0,2,1]
```

(reversed)

---

### Step 2: Count Peaks & Valleys

For every middle digit:

```text
prev current next
```

Check:

#### Peak

```text
prev < current > next
```

#### Valley

```text
prev > current < next
```

Example:

```text
120

[0,2,1]

0 < 2 > 1

Peak Found
```

---

### Step 3: Generate Next Number

Instead of:

```text
120 → convert
121 → convert
122 → convert
```

reuse the existing digit array:

```text
120
↓
121
↓
122
↓
123
```

using manual carry propagation.

Example:

```text
129

[9,2,1]

+1

[0,3,1]
```

which represents:

```text
130
```

## ✅ Why Solution Works

For every number:

* Every middle digit is checked exactly once.
* Peak and valley definitions are applied directly.
* All numbers in the range are processed.

Therefore:

* Every valid waviness contribution is counted.
* No invalid contribution is counted.

Thus the algorithm always returns the correct total waviness.

---

## 💻 Code

```ts
function totalWaviness(num1: number, num2: number): number {

    function getNextNum(numArray: number[]): void {
        let carry = 1;

        for(let i = 0; i < numArray.length; i++){
            const num = numArray[i] + carry;

            numArray[i] = num % 10;
            carry = Math.floor(num / 10);
        }

        if(carry !== 0){
            numArray.push(carry);
        }
    }

    function getWavinessCount(numArray: number[]): number {
        let count = 0;

        for(let i = 1; i < numArray.length - 1; i++){
            const prev = numArray[i - 1];
            const curr = numArray[i];
            const next = numArray[i + 1];

            if(prev < curr && curr > next){
                count++;
            }
            else if(prev > curr && curr < next){
                count++;
            }
        }

        return count;
    }

    function getNumArray(num: number): number[] {
        const digits: number[] = [];

        while(num > 0){
            digits.push(num % 10);
            num = Math.floor(num / 10);
        }

        return digits;
    }

    function countWaviness(startNum: number, count: number): number {
        let waviness = 0;

        const digits = getNumArray(startNum);

        waviness += getWavinessCount(digits);

        for(let i = 0; i < count; i++){
            getNextNum(digits);
            waviness += getWavinessCount(digits);
        }

        return waviness;
    }

    return countWaviness(num1, num2 - num1);
}
```

## ⏱️ Complexity Analysis

| Operation | Complexity                      |
| --------- | ------------------------------- |
| Time      | `O((num2 - num1 + 1) × digits)` |
| Space     | `O(digits)`                     |

Since:

```text
digits ≤ 6
```

the solution is effectively linear in the size of the range.

## 🎯 Pattern Recognition

This problem teaches an important lesson:

### Not Every Digit Problem Needs Digit DP

Signal:

```text
Range Query
+
Digit Property
```

Normally suggests:

```text
Digit DP
```

But constraints matter.

Because:

```text
num2 ≤ 100000
```

a direct simulation is already fast enough.

---

### Future Signal

If tomorrow's version becomes:

```text
num2 ≤ 10^18
```

then this approach immediately breaks.

At that point the correct pattern becomes:

```text
Digit DP
```

because iterating through every number is impossible.

---

## 📝 Key Takeaways

* Always inspect constraints before designing an algorithm.
* Digit properties often suggest Digit DP, but small constraints may allow simulation.
* Reusing a digit array is a neat optimization over repeatedly converting numbers.
* Peaks and valleys only depend on neighboring digits.
* Sometimes the intended solution is much simpler than the problem statement suggests.

---

## ⚠️ Common Mistakes

### Forgetting Edge Digits

```text
First digit
Last digit
```

can never be peaks or valleys.

---

### Using String Conversion for Every Number

```text
120
121
122
...
```

works, but repeatedly rebuilding digit arrays does unnecessary work.

---

### Missing Valley Condition

Checking only:

```text
prev < curr > next
```

and forgetting:

```text
prev > curr < next
```

leads to incorrect answers.

---

### Overengineering Into Digit DP

The problem looks like a Digit DP problem, but the constraints are small enough that simulation is sufficient.
