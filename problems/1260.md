# 🧩 Problem #1260 - Shift 2D Grid

**Difficulty:** Easy    
**Topics:** `Matrix`, `Simulation`, `Math`      
**Link:** https://leetcode.com/problems/shift-2d-grid/     

---

# 💡 Intuition

The shifting rules may initially seem like a matrix problem:

* Move right within a row.
* Wrap to the next row.
* Wrap the last element back to the beginning.

Instead of handling all these edge cases separately, notice that the grid is stored in **row-major order**.

For example,

```text
1 2 3
4 5 6
7 8 9
```

can be viewed as

```text
[1,2,3,4,5,6,7,8,9]
```

Now the problem simply becomes:

> **Rotate a 1D array to the right by `k` positions.**

There is also another elegant solution that never explicitly flattens the grid. Instead, it treats every cell as its **linear index** and computes its new position directly.

# 🔍 Key Observation

A matrix stored row-by-row is equivalent to a single linear array.

For a matrix

```text
1 2 3
4 5 6
7 8 9
```

its linear representation is

```text
[1,2,3,4,5,6,7,8,9]
```

Every shift operation is simply a circular shift of this linear sequence.

# 🧠 Attempt 1 — Flatten → Rotate → Rebuild

## Idea

Convert the matrix into a linear array.

Instead of physically rotating the array, begin reading from

```text
totalElements - shiftBy
```

and wrap around using modular arithmetic.

Finally, rebuild the matrix row-by-row.

### Your Solution

```ts
function shiftGrid(grid: number[][], k: number): number[][] {
    const n = grid.length;
    const m = grid[0].length;
    const totalElements = n * m;
    const shiftBy = k % totalElements;

    if (shiftBy === 0) return grid;

    const elements = [];

    for (let i = 0; i < n; i++) {
        for (let j = 0; j < m; j++) {
            elements.push(grid[i][j]);
        }
    }

    let index = totalElements - shiftBy;

    for (let i = 0; i < n; i++) {
        for (let j = 0; j < m; j++) {
            grid[i][j] = elements[index];
            index++;

            if (index === totalElements)
                index = 0;
        }
    }

    return grid;
}
```

## Why It Works

Flattening preserves the row-major ordering of the matrix.

Reading the flattened array from

```text
totalElements - shiftBy
```

is equivalent to performing a right circular rotation.

Reconstructing row-by-row restores the correct matrix.

# 🧠 Attempt 2 — Direct Index Mapping (Without Flattening)

## Idea

Instead of creating a temporary linear array, observe that every matrix cell already has a **linear index**.

For a matrix with

* `rows`
* `cols`

the cell

```text
(row, col)
```

corresponds to

```text
index = row × cols + col
```

After shifting,

```text
newIndex = (index + k) % totalElements
```

Convert this new index back into matrix coordinates.

```text
newRow = Math.floor(newIndex / cols)

newCol = newIndex % cols
```

Place the current value directly into its new position.

---

## Solution

```ts
function shiftGrid(grid: number[][], k: number): number[][] {
    const rows = grid.length;
    const cols = grid[0].length;

    const totalElements = rows * cols;
    const shiftBy = k % totalElements;

    if (shiftBy === 0) return grid;

    const result = Array.from(
        { length: rows },
        () => Array(cols)
    );

    for (let row = 0; row < rows; row++) {
        for (let col = 0; col < cols; col++) {

            const index = row * cols + col;

            const newIndex = (index + shiftBy) % totalElements;

            const newRow = Math.floor(newIndex / cols);
            const newCol = newIndex % cols;

            result[newRow][newCol] = grid[row][col];
        }
    }

    return result;
}
```

---

## Why It Works

Every element has exactly one linear position.

Instead of moving values one shift at a time, compute their final destination directly.

The mapping

```text
(row, col)

↓

linear index

↓

shift

↓

new linear index

↓

(newRow, newCol)
```

guarantees that every value is placed exactly where it belongs after all `k` shifts.

| **2. Direct Index Mapping**       | Compute each element's final position directly | `O(m × n)` | `O(m × n)` | Avoids explicitly flattening the matrix |

# ✅ Why Modulo is Important

Suppose

```text
9 elements

k = 18
```

Shifting by

```text
18
```

is exactly the same as shifting by

```text
18 % 9 = 0
```

Therefore,

```text
shiftBy = k % totalElements
```

eliminates unnecessary work.

---

# ⚖️ Why it Works

A row-major matrix is simply another representation of a linear array.

Both approaches exploit this fact:

* The first explicitly converts the matrix into a linear sequence.
* The second computes the equivalent linear indices mathematically.

Since every element's new position after a shift is uniquely determined, both methods produce the same final grid.

# 🏆 Key Takeaways

* Many matrix problems become easier after converting them into **linear index problems**.
* Row-major traversal provides a natural mapping between 2D coordinates and a 1D array.
* Circular shifts almost always involve **modular arithmetic**.
* Instead of simulating each shift individually, compute the final position directly.
