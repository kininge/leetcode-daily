# 🧩 Problem #3043 – Find the Length of the Longest Common Prefix

**Difficulty:** Medium
**Topics:** `Array`, `Hash Map`, `String`, `Trie`
**Link:** [https://leetcode.com/problems/find-the-length-of-the-longest-common-prefix/](https://leetcode.com/problems/find-the-length-of-the-longest-common-prefix/)

---

## 📝 Problem Summary

You are given two arrays:

```txt
arr1
arr2
```

containing positive integers.

A prefix of a number consists of digits starting from the leftmost side.

Example:

```txt
12345
```

has prefixes:

```txt
1
12
123
1234
12345
```

Your task is to find:

```txt
maximum length of common prefix
```

between any pair:

```txt
x ∈ arr1
y ∈ arr2
```

If no common prefix exists:

```txt
return 0
```

---

### Example

Input:

```txt
arr1 = [1,10,100]
arr2 = [1000]
```

Possible common prefixes:

```txt
1
10
100
```

Longest common prefix:

```txt
100
```

Length:

```txt
3
```

---

### Constraints

* `1 <= arr1.length, arr2.length <= 5 * 10^4`
* `1 <= arr1[i], arr2[i] <= 10^8`

---

## 💡 Intuition

At first glance, this problem looks like:

```txt
compare every pair of numbers
```

and compute longest common prefix.

However:

```txt
5 * 10^4 × 5 * 10^4
```

pair comparisons are far too large.

The key optimization is realizing:

```txt
prefixes can be reused
```

instead of recalculated repeatedly.

---

## 🛠 Approach 1 — Pairwise Prefix Comparison (TLE)

### Initial Thought Process

A natural first idea is:

For every pair:

```txt
(num1, num2)
```

* compare digits from left to right
* count matching prefix digits
* track maximum answer

---

### Step 1 — Convert Numbers Into Digits

To compare prefixes digit-by-digit:

```txt
12345 → [1,2,3,4,5]
```

Digit arrays are generated for each number.

---

### Step 2 — Compare Every Pair

For every:

```txt
num1 ∈ arr1
num2 ∈ arr2
```

compare corresponding digits until mismatch occurs.

---

### Why This Gives TLE

Nested loops already produce:

```txt
O(n × m)
```

Worst case:

```txt
5 * 10^4 × 5 * 10^4
≈
2.5 billion comparisons
```

Even though:

```txt
maximum digits <= 9
```

pairwise iteration becomes far too expensive.

The issue is:

```txt
same prefixes are repeatedly recomputed
```

across many comparisons.

---

### Code (TypeScript)

```ts
function getDigitsArray(num) {
    const digits = [];

    let _num = num;

    while (_num > 0) {

        const reminder = _num % 10;

        _num = Math.floor(_num / 10);

        digits.push(reminder);
    }

    return digits;
}

function longestCommonPrefix(
    arr1: number[],
    arr2: number[]
): number {

    let longestPrefixNumSize = 0;

    // avoid repeated digit conversions
    const store = new Map();

    for (const num of arr1) {

        if (!store.has(num)) {
            store.set(num, getDigitsArray(num));
        }
    }

    for (const num of arr2) {

        if (!store.has(num)) {
            store.set(num, getDigitsArray(num));
        }
    }

    // O(n² * digits)
    for (const num1 of arr1) {

        for (const num2 of arr2) {

            const num1Digits = store.get(num1);
            const num2Digits = store.get(num2);

            const n = num1Digits.length;
            const m = num2Digits.length;

            // exact same number
            if (num1 === num2) {
                longestPrefixNumSize = Math.max(
                    longestPrefixNumSize,
                    n
                );

                break;
            }

            const minDigits = Math.min(n, m);

            let count = 0;

            for (let i = 0; i < minDigits; i++) {

                // arrays store digits reversed
                if (
                    num1Digits[n - i - 1] ===
                    num2Digits[m - i - 1]
                ) {
                    count++;
                }
                else {
                    break;
                }
            }

            longestPrefixNumSize = Math.max(
                longestPrefixNumSize,
                count
            );
        }
    }

    return longestPrefixNumSize;
}
```

---

## 🔥 Key Optimization Insight

Instead of:

```txt
comparing every pair repeatedly
```

we can:

```txt
precompute all prefixes once
```

and perform fast membership lookups.

This converts repeated comparison work into:

```txt
HashSet lookup problem
```

---

## 🛠 Approach 2 — Store All Prefixes (Optimal)

### Idea

For every number in:

```txt
arr1
```

store all possible prefixes.

Example:

```txt
12345
```

produces:

```txt
12345
1234
123
12
1
```

Then for every number in:

```txt
arr2
```

check prefixes from longest to shortest.

The first matching prefix automatically becomes:

```txt
longest common prefix for that number
```

---

### Step 1 — Store All Prefixes Of arr1

Repeatedly remove last digit:

```ts
_num = Math.floor(_num / 10)
```

while inserting prefixes into a Set.

---

### Step 2 — Search Prefixes Of arr2

Again repeatedly remove last digit.

If prefix exists in Set:

```txt
common prefix found
```

Since we shrink from longest to shortest:

```txt
first match is optimal
```

for that number.

---

## 🔥 Smart Optimization

This line is important:

```ts
break;
```

Once a prefix is found:

```txt
no need to check smaller prefixes
```

because current prefix is already longest possible.

---

### Why It Works

Every valid prefix from `arr1` is precomputed once.

Now instead of repeated pairwise comparisons:

```txt
membership lookup becomes O(1)
```

This removes the expensive nested-pair processing.

---

### Code (TypeScript)

```ts
function getDigitsCouns(num): number {

    let _num: number = num;

    let count: number = 0;

    while (_num > 0) {

        count++;

        _num = Math.floor(_num / 10);
    }

    return count;
}

function longestCommonPrefix(
    arr1: number[],
    arr2: number[]
): number {

    const arr1AllPossiblePrefix = new Set<number>();

    // generate all prefixes from arr1
    for (const num of arr1) {

        let _num = num;

        while (_num > 0) {

            arr1AllPossiblePrefix.add(_num);

            _num = Math.floor(_num / 10);
        }
    }

    let longestPrefixNumSize = 0;

    // search prefixes for arr2
    for (const num of arr2) {

        let _num = num;

        while (_num > 0) {

            if (arr1AllPossiblePrefix.has(_num)) {

                const digitsSize = getDigitsCouns(_num);

                longestPrefixNumSize = Math.max(
                    longestPrefixNumSize,
                    digitsSize
                );

                break;
            }

            _num = Math.floor(_num / 10);
        }
    }

    return longestPrefixNumSize;
}
```

## 🛠 Approach 3 — Trie (Prefix Tree)

### Idea

This problem can also be solved using a:

```txt
Trie / Prefix Tree
```

Insert all numbers from:

```txt
arr1
```

into Trie digit-by-digit.

Then for each number in:

```txt
arr2
```

traverse Trie while digits match.

Traversal depth becomes longest common prefix length.

---

### Why Trie Is Less Preferred Here

Trie is conceptually elegant for prefix problems.

However:

* implementation is longer
* HashSet solution is simpler
* constraints are small enough
* both approaches are effectively linear here

So Set-based prefix storage is usually preferable.

---

## ⚖️ Approach Comparison

| Approach            | Time Complexity       | Auxiliary Space | Notes                          |
| ------------------- | --------------------- | --------------- | ------------------------------ |
| Pairwise Comparison | `O(n × m × digits)`   | `O(n + m)`      | Gives TLE                      |
| Prefix HashSet      | `O((n + m) × digits)` | `O(n × digits)` | Optimal practical solution     |
| Trie                | `O((n + m) × digits)` | `O(n × digits)` | More advanced prefix structure |

---

## ⏱ Complexity Analysis

### Prefix HashSet Approach

Maximum digits:

```txt
<= 9
```

So digit operations behave like constant time.

| Operation         | Complexity |
| ----------------- | ---------- |
| Generate prefixes | `O(9 × n)` |
| Query prefixes    | `O(9 × m)` |
| Total Time        | `O(n + m)` |
| Auxiliary Space   | `O(n × 9)` |

## 📌 Key Takeaways

* Repeated pairwise comparison often signals optimization opportunity.

* Precomputing reusable information can drastically reduce complexity.

* Prefix problems frequently relate to:

```txt
HashSet
Trie
Prefix Tree
```

* Removing digits using:

```ts
Math.floor(num / 10)
```

is an elegant numeric way to generate prefixes.

* Understanding why an approach gives TLE is extremely valuable for future pattern recognition.

## ⚠️ Common Mistakes

### 1️⃣ Comparing Every Pair Directly

This leads to:

```txt
O(n × m)
```

behavior which is too large.

---

### 2️⃣ Recomputing Prefixes Repeatedly

Same prefixes may appear in many comparisons.

Precompute once instead.

---

### 3️⃣ Forgetting Early Break

Once longest prefix for a number is found:

```txt
smaller prefixes are unnecessary
```

---

### 4️⃣ Overengineering With Trie

Trie works well but may be unnecessary when HashSet solution is simpler and equally efficient.
