# 🧩 Problem #2492 – Minimum Score of a Path Between Two Cities

**Difficulty:** Medium  
**Topics:** `Graph`, `Union Find`   
**Link:** https://leetcode.com/problems/minimum-score-of-a-path-between-two-cities/    

---

## 💡 Intuition

At first, this problem looks like a shortest path problem.

But one sentence completely changes everything:

> **It is allowed for a path to contain the same road multiple times.**

This means we are **not** looking for an optimal path.

Since we can revisit roads, once cities **1** and **n** belong to the same connected component, we can freely travel anywhere inside that component before finally reaching city **n**.

So the answer is simply:

```text
Smallest road distance inside the connected component
containing city 1.
```

No shortest path algorithm is needed.

## ✅ My Solution

```ts
// if we can choose any path not required optimal
// then ans is --> smallest distance between connectd group where city 1 an dn both exist
function minScore(n: number, roads: number[][]): number {
    
    // edge case
    if(n === 2) return roads[0][2];

    // no need to build graph at all
    const unionFind = new MyUnionFind(n+1);

    // O(m)
    for(const [node1, node2] of roads){
        unionFind.union(node1, node2);
    }

    // find minimum road inside connected component
    let minDistanceRoad = Infinity;

    // O(m)
    for(const [node1, node2, distance] of roads){
        if(unionFind.connected(1, node1)){
            minDistanceRoad = Math.min(minDistanceRoad, distance);
        }
    }

    return minDistanceRoad;
}

/**
 * Disjoint Set Union (Union Find)
 *
 * Features
 * --------
 * ✅ Path Compression
 * ✅ Union by Size
 *
 * Complexity
 * ----------
 * find()          : O(α(n))
 * union()         : O(α(n))
 * connected()     : O(α(n))
 * componentSize() : O(α(n))
 *
 * α(n) = Inverse Ackermann Function
 * (Practically constant time.)
 */
class MyUnionFind {
    private _size: number[];
    private _root: number[];
    private _components: number;

    constructor(nodesInGraph: number){
        this._components = nodesInGraph;
        this._size = new Array(nodesInGraph).fill(1);
        this._root = [];
        for(let i=0; i<nodesInGraph; i++) this._root.push(i);
    }

    find(node: number): number {
        if(node === this._root[node]) return node;

        const root = this.find(this._root[node]);
        this._root[node] = root;

        return root;
    }

    union(node1: number, node2: number): boolean {
        const rootNode1 = this.find(node1);
        const rootNode2 = this.find(node2);

        const unionSize1 = this._size[rootNode1];
        const unionSize2 = this._size[rootNode2];

        if(rootNode1 === rootNode2) return false;

        if(unionSize1 >= unionSize2){
            this._size[rootNode1] += unionSize2;
            this._root[rootNode2] = rootNode1;
        }else{
            this._size[rootNode2] += unionSize1;
            this._root[rootNode1] = rootNode2;
        }

        this._components--;
        return true;
    }

    connected(node1: number, node2: number): boolean {
        return this.find(node1) === this.find(node2);
    }

    componentSize(node: number): number {
        return this._size[this.find(node)];
    }

    components(): number {
        return this._components;
    }
}
```

## 🔄 Alternative Approach

### My Approach (Union Find)

| Metric | Value |
| ------- | ----- |
| Time | O(n + m α(n)) |
| Space | O(n) |

### Alternative (DFS / BFS)

| Metric | Value |
| ------- | ----- |
| Time | O(n + m) |
| Space | O(n) |

Start DFS/BFS from city **1**.

Visit every reachable city while tracking the minimum edge encountered.

Both approaches are optimal.

---

## ⚖️ Approach Comparison

| Approach | Time | Space | Notes |
| -------- | ---- | ----- | ----- |
| Union Find | O(n + m α(n)) | O(n) | Best when connectivity queries are useful |
| DFS | O(n+m) | O(n) | Simple graph traversal |
| BFS | O(n+m) | O(n) | Also works |

## ⏱️ Complexity Analysis

Let


```text

O(m)

O(n + m α(n))
```

Since α(n) (Inverse Ackermann Function) grows extremely slowly,

it is practically constant.

