# 🧩 Problem #1967 – Number of Strings That Appear as Substrings in Word

**Difficulty:** Easy   
**Topics:** `String`   
**Link:** https://leetcode.com/problems/number-of-strings-that-appear-as-substrings-in-word/   

---

## 💡 Intuition

For every pattern, simply check whether it appears anywhere inside the given word.

Since the constraints are very small:

```text
patterns.length <= 100
word.length <= 100
```

a straightforward solution is sufficient.

The idea is:

```text
Pattern
    ↓
Scan Entire Word
    ↓
Found?
    ↓
Count++
```

## ✅ My Solution

```ts
function numOfStrings(patterns: string[], word: string): number {
    let count = 0;
    const n = word.length;

    for(const wordCheck of patterns){
        const _n = wordCheck.length;

        if(_n > n) continue;

        for(let i=0; i<n; i++){
            if(word[i] === wordCheck[0]){
                const compare = word.substring(i, i+_n);

                if(wordCheck === compare){
                    count++;
                    break;
                }
            }
        }
    }

    return count;
};
```

## 🔄 Alternative Approach

### My Approach

| Metric | Value        |
| ------ | ------------ |
| Time   | O(m × n × k) |
| Space  | O(1)         |

where

```text
m = patterns.length
n = word.length
k = average pattern length
```

The substring creation and comparison each depend on the pattern length.

---

### Alternative Approaches (For Learning)

#### KMP (Knuth-Morris-Pratt)

Suitable when searching:

```text
One Pattern

↓

One Large Text
```

Time:

```text
O(n + k)
```

per pattern.

---

#### Trie

Useful when searching:

```text
Many Patterns
```

A Trie can store all patterns together and search from every starting position of the word.

Although educational, it is unnecessary for this problem because the constraints are already very small.

---

#### Aho-Corasick

Designed specifically for:

```text
Many Patterns

↓

One Large Text
```

Time:

```text
O(totalTextLength + totalPatternLength)
```

This algorithm combines:

```text
Trie
      +
Failure Links
```

and avoids restarting the search from the Trie root.

## ⚖️ Approach Comparison

| Approach     | Time           | Space | Notes                     |
| ------------ | -------------- | ----- | ------------------------- |
| Brute Force  | O(m × n × k)   | O(1)  | Simple and sufficient     |
| KMP          | O(m × (n + k)) | O(k)  | Better for one pattern    |
| Trie         | O(P + n²)      | O(P)  | Good learning exercise    |
| Aho-Corasick | O(P + n)       | O(P)  | Optimal for many patterns |

where:

```text
P = total characters across all patterns
```

## ⏱️ Complexity Analysis

Let:

```text
m = patterns.length

n = word.length

k = average pattern length
```

### Time Complexity

```text
O(m × n × k)
```

For every pattern:

* Scan the word.
* Create a substring.
* Compare it with the pattern.

Since:

```text
k ≤ n
```

this is sometimes simplified as:

```text
O(m × n²)
```

---

### Space Complexity

```text
O(1)
```

Only a few variables are used.
