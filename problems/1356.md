# 🧩 Problem #1356 – Sort Integers by The Number of 1 Bits

**Difficulty:** Easy  
**Topics:**  `Array`, `Sorting`, `Bit Manipulation`  
**Link:** https://leetcode.com/problems/sort-integers-by-the-number-of-1-bits/

---

## 📝 Problem Summary

You are given an integer array `arr`.

Your task is to sort the array based on:

1. **Primary condition:** Number of `1`s in the binary representation (ascending)
2. **Secondary condition:** Numeric value itself (ascending)

Return the sorted array.

---

### Example

Input: `arr = [0,1,2,3,4,5,6,7,8]`
Binary representations:
```txt
0 → 0000 (0 ones)
1 → 0001 (1 one)
2 → 0010 (1 one)
3 → 0011 (2 ones)
4 → 0100 (1 one)
5 → 0101 (2 ones)
6 → 0110 (2 ones)
7 → 0111 (3 ones)
8 → 1000 (1 one)
```
Sorted result: `[0,1,2,4,8,3,5,6,7]`

---

### Constraints

- `1 <= arr.length <= 500`
- `0 <= arr[i] <= 10^4`

---

### Key Observations

- Maximum value is `10^4`
- `10^4` in binary needs **14 bits**
- So counting set bits requires checking at most 14 positions
- Sorting dominates overall complexity (`O(n log n)`)

---

## 💡 Intuition

We must sort numbers based on two criteria:

1. Count of `1` bits (ascending)
2. Numeric value (ascending if tie)

So the real problem becomes:

> How efficiently can we compute the number of set bits for each number?

---

### Observations

- Each number is at most `10^4`
- Binary length ≤ 14 bits
- So counting set bits is inexpensive

Since sorting is required, lower bound is: `O(nlogn)`

Bit counting per number is small constant work.

---

### Strategy

1. For each number:
   - Count number of `1` bits
2. Sort using:
   - Primary key → bit count
   - Secondary key → number itself

The key design question is:

- Do we precompute bit counts?
- Or compute inside comparator?
- Or use a better bit counting technique?

We'll start with a straightforward implementation.

---

## 🛠 Approach 1 — Count Bits Manually + Custom Sort

### Idea

1. For each number in `arr`:
   - Count the number of set bits (`1`s) using bit masking.
   - Since `arr[i] ≤ 10^4`, we only need to check **14 bits**.
2. Store `{ num, count }` pairs.
3. Sort:
   - First by `count`
   - Then by `num`
4. Extract sorted numbers back into result array.

This approach separates bit counting from sorting logic.

---

### Why it works

- Each number can be uniquely ranked by:
  - Its bit count
  - Its numeric value
- Sorting by these two keys guarantees correct order.
- Counting 14 bits per number is constant time.

---

### Code (TypeScript)

```ts
function sortByBits(arr: number[]): number[] {
    const numAndCounts: { num: number; count: number }[] = [];

    // Count bits (at most 14 bits needed for 10^4)
    for (let i = 0; i < arr.length; i++) {
        let count = 0;

        for (let j = 0; j < 14; j++) {
            if ((arr[i] & (1 << j)) !== 0) {
                count++;
            }
        }

        numAndCounts.push({ num: arr[i], count });
    }

    // Sort by bit count, then by numeric value
    numAndCounts.sort((a, b) => {
        if (a.count !== b.count) {
            return a.count - b.count;
        }
        return a.num - b.num;
    });

    // Extract sorted numbers
    const result: number[] = [];
    for (const item of numAndCounts) {
        result.push(item.num);
    }

    return result;
}
```

---

### Complexity

- **Time Complexity:**
  - Bit counting: `O(n * 14)` → `O(n)`
  - Sorting: `O(n log n)`
  - Overall: `O(n log n)`
- **Space Complexity:**
  - Auxiliary array storing `{num, count}` pairs → `O(n)`

---

## 🚀 Approach 2 — Use Efficient Bit Counting + Direct Sort (Improved)

### Idea

Instead of:

- Manually checking all 14 bit positions
- Storing `{num, count}` pairs in a separate array

We can:

1. Use a more efficient bit counting technique.
2. Sort the array directly using a comparator.

A better bit counting method is **Brian Kernighan’s Algorithm**:
```txt
  while (n > 0) {
    n &= (n - 1);
    count++;
  }
```

Each iteration removes the lowest set bit.

So complexity depends on number of `1`s, not total bits.

---

### Why it works

Property: `n & (n - 1)`

removes the rightmost set bit.

Example:
```
n = 12 (1100)
n-1 = 11 (1011)

    1100
  & 1011
  ------
    1000 (lowest 1 removed)
```

This continues until `n` becomes `0`.

So total iterations = number of `1`s.

---

### Code (TypeScript)

```ts
function sortByBits(arr: number[]): number[] {

    const countBits = (n: number): number => {
        let count = 0;
        while (n > 0) {
            n &= (n - 1);
            count++;
        }
        return count;
    };

    return arr.sort((a, b) => {
        const bitA = countBits(a);
        const bitB = countBits(b);

        if (bitA !== bitB) return bitA - bitB;
        return a - b;
    });
}
```

---

### Complexity

- **Time Complexity:**
  - Bit counting: proportional to number of set bits
  - Sorting: `O(n log n)`
  - Overall: `O(n log n)`
- **Space Complexity:**
  - Sorting in place → `O(1)` (ignoring recursion stack of sort)

---

### Why this is better than Approach 1

- No extra `{num, count}` array
- Cleaner comparator logic
- Bit counting depends only on actual set bits
- More interview-impressive (shows bit manipulation knowledge)

This is the preferred interview solution.

---

## 📊 Comparison of Approaches

| Criteria | 🛠 Approach 1 — Manual 14-Bit Count + Pair Array | 🚀 Approach 2 — Kernighan + Direct Sort |
|-----------|--------------------------------------------------|------------------------------------------|
| Bit Counting Method | Check all 14 bit positions | Remove lowest set bit each iteration |
| Counting Cost per Number | Always 14 iterations | Depends on number of `1`s |
| Extra Data Structure | Yes (`{num, count}` array) | No |
| Sorting Strategy | Sort auxiliary array | Sort original array directly |
| Time Complexity | O(n log n) | O(n log n) |
| Space Complexity | O(n) | O(1) (in-place sort) |
| Code Cleanliness | Moderate | Cleaner |
| Interview Strength | Good | Stronger (bit trick knowledge) |

---

### 🏆 Final Recommendation

- Approach 1 is straightforward and easy to reason about.
- Approach 2 is cleaner and demonstrates stronger understanding of bit manipulation.

For interviews,  
**Approach 2 (Kernighan + direct comparator) is preferred.**
