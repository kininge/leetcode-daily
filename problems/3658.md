## 💡 Intuition

The problem asks us to compute

* `sumOdd` = sum of the first `n` odd numbers
* `sumEven` = sum of the first `n` even numbers

and then return

```text
GCD(sumOdd, sumEven)
```

A straightforward approach is to compute both sums directly and then apply the Euclidean Algorithm.

However, this problem hides a beautiful mathematical observation that reduces the entire solution to **one line**.

# 🧠 Attempt 1 — Simulation + Euclidean Algorithm

## Idea

Iterate through the first `n` odd and even numbers.

Compute both sums.

Finally compute their GCD.

### Your Solution

```ts
function gcdOfOddEvenSums(n: number): number {
    let evenSum = 0;
    let oddSum = 0;

    for (let i = 1; i <= 2 * n; i += 2) {
        oddSum += i;
        evenSum += i + 1;
    }

    function GCD(a: number, b: number): number {
        if (b === 0) return a;
        return GCD(b, a % b);
    }

    return GCD(
        Math.max(evenSum, oddSum),
        Math.min(evenSum, oddSum)
    );
}
```

---

### Complexity

Summation

```text
O(n)
```

GCD

```text
O(log(max(sum)))
```

Overall

```text
O(n)
```

# 🧠 Attempt 2 — Mathematical Observation

Instead of computing the sums, derive them.

---

## Deriving the Formula

### Sum of First n Natural Numbers

We know

```text
S = 1 + 2 + 3 + ... + n
```

Write it in reverse.

```text
S = 1 + 2 + 3 + ... + n

S = n + (n-1) + ... + 2 + 1
```

Add both equations.

```text
2S = (n+1) + (n+1) + ... + (n+1)
```

There are exactly `n` terms.

Therefore,

```text
2S = n(n+1)
```

Hence,

```text
S = n(n+1)/2
```

---

### Sum of First n Even Numbers

The sequence is

```text
2 + 4 + 6 + ... + 2n
```

Factor out `2`.

```text
= 2(1 + 2 + ... + n)
```

Substitute the previous formula.

```text
= 2 × n(n+1)/2
```

The `2` cancels.

```text
sumEven = n(n+1)
```

---

### Sum of First n Odd Numbers

Let's compute a few examples.

```text
1 = 1²

1 + 3 = 4 = 2²

1 + 3 + 5 = 9 = 3²

1 + 3 + 5 + 7 = 16 = 4²
```

This suggests

```text
sumOdd = n²
```

#### Proof

Assume

```text
S(n-1) = (n-1)²
```

Then

```text
S(n)
=
S(n-1)
+
(2n-1)
```

Substitute.

```text
=
(n-1)² + (2n-1)
```

Expand.

```text
=
n² - 2n + 1 + 2n - 1

=
n²
```

Therefore,

```text
1 + 3 + 5 + ... + (2n-1)

=

n²
```

---

## Simplifying the GCD

Now substitute the formulas.

```text
sumOdd = n²

sumEven = n(n+1)
```

The problem becomes

```text
GCD(n², n(n+1))
```

Factor out `n`.

```text
= n × GCD(n, n+1)
```

Since consecutive integers are always coprime,

```text
GCD(n, n+1) = 1
```

Therefore,

```text
Answer = n
```

---

## ✅ Solution

```ts
function gcdOfOddEvenSums(n: number): number {
    return n;
}
```

---

## ⚖️ Why it Works

Using the derived formulas,

```text
sumOdd = n²

sumEven = n(n+1)
```

Their GCD becomes

```text
GCD(n², n(n+1))
```

Factoring out the common term,

```text
= n × GCD(n, n+1)
```

Since any two consecutive integers share no common divisor greater than `1`,

```text
GCD(n, n+1) = 1
```

Hence,

```text
GCD(sumOdd, sumEven)

=

n
```

## 🏆 Key Takeaways

* Before computing a sequence sum, ask whether a mathematical formula already exists.
* The Euclidean Algorithm relies on the identity

```text
GCD(a, b)

=

GCD(b, a % b)
```

* Consecutive integers are always coprime.

```text
GCD(n, n+1) = 1
```

* Combining mathematical identities can often eliminate iteration entirely.
