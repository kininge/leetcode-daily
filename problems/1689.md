# 🧩 Problem #1689 – Partitioning Into Minimum Number Of Deci-Binary Numbers

**Difficulty:** Medium  
**Topics:**  `Greedy`, `String`, `Math`  
**Link:** https://leetcode.com/problems/partitioning-into-minimum-number-of-deci-binary-numbers/

---

## 📝 Problem Summary

A **deci-binary number** is a decimal number whose digits are only `0` or `1`, and it has no leading zeros.

You are given a decimal string `n`.  
Return the **minimum number of deci-binary numbers** required such that their sum equals `n`.

---

### Example

Input: ``
Output: 3

Explanation:

We can represent 32 as:
```txt
    11
  + 11
  + 10
  ----
    32
```

We needed 3 deci-binary numbers.

---

### Constraints

- `1 <= n.length <= 10^5`
- `n` consists only of digits (`0–9`)
- No leading zeros
- `n` represents a positive integer

---

### Key Observation

Each deci-binary number contributes **at most 1 per digit position**.

So for any digit `d` in `n`,  
we need at least `d` deci-binary numbers to build that column.

Therefore, the answer depends only on the **maximum digit** in `n`.

---

## 💡 Intuition

Think about the addition **column-wise**.

Each deci-binary number:

- Has only `0` or `1` in every digit position.
- So in any column, it can contribute at most `1`.

If a digit in `n` is: `d`

Then to build that column:

- We need at least `d` deci-binary numbers.
- Because the maximum contribution per number per column is `1`.

---

### Example
n = `82734`
Digits: `8 2 7 3 4`

The first digit is `8`.

To construct that column:
- We must stack **8 ones** vertically.
- So we need at least 8 deci-binary numbers.

Other digits may require fewer,
but the largest digit determines the minimum count.

---

### Core Insight

There is **no carry dependency between columns**.

Each digit can be handled independently.

So the minimum number of deci-binary numbers required is: `maximum digit in n`

That’s it.

No DP.
No BFS.
No simulation.

Pure greedy digit observation.

---

## 🛠 Approach 1 — Greedy (Maximum Digit Observation)

### Idea

Each deci-binary number can contribute **at most 1** to any digit position.

Therefore:

- If a digit in `n` is `d`,
- We need at least `d` deci-binary numbers to construct that column.

Since all columns are independent, the minimum number of deci-binary numbers required is simply: `max digit in n`

---

### Why it works

- Deci-binary numbers contain only `0` or `1`.
- When summing multiple such numbers, each column is just the sum of 0s and 1s.
- No column can exceed the number of deci-binary numbers used.
- Therefore, the largest digit determines the minimum required count.
- This count is always sufficient (constructible by stacking 1s column-wise).

---

### Code (TypeScript)

```ts
function minPartitions(n: string): number {
    let maxDigit = 0;

    for (let i = 0; i < n.length; i++) {
        const digit = n.charCodeAt(i) - 48; // faster than Number(n[i])
        maxDigit = Math.max(maxDigit, digit);

        // Early exit: cannot exceed 9
        if (maxDigit === 9) break;
    }

    return maxDigit;
}
```

### Complexity

- **Time Complexity:**
  - Single pass through string → `O(n)`
- **Space Complexity:**
  - Constant variables only → `O(1)`

---

### 🏆 Final Recommendation

- The greedy maximum-digit observation is both necessary and sufficient.
- No dynamic programming or simulation is required.
- Optimal in both time and space.

This is a classic example where recognizing digit independence avoids overengineering.
