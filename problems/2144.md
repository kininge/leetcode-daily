# 🧩 Problem #2144 – Minimum Cost of Buying Candies With Discount

**Difficulty:** Easy    
**Topics:** `Greedy`, `Sorting`  
**Link:** https://leetcode.com/problems/minimum-cost-of-buying-candies-with-discount/  

---

# 📖 Problem Summary

For every:

```txt
2 candies purchased
```

we can get:

```txt
1 candy free
```

Constraint:

```txt
freeCandy <= cheaperPurchasedCandy
```

We need to buy all candies with minimum total cost.

---

# 💡 Intuition

To minimize total cost:

```txt
We want the free candy
to be as expensive as possible.
```

Since the free candy must be:

```txt
<= both purchased candies
```

a natural idea is:

```txt
Sort candies
```

and try to form groups where:

```txt
Most expensive candies are paid
Third expensive candy becomes free
```

---

# 🔍 Key Observation

Suppose after sorting:

```txt
[1,2,3,4,5,6]
```

Process from largest:

```txt
6,5,(4 free)
3,2,(1 free)
```

Cost:

```txt
6 + 5 + 3 + 2
```

Notice:

```txt
Every 3rd candy
can be made free.
```

after sorting in descending order.

---

# 🛠️ Approach 1 (My Approach)

Sort the array.

Process from largest cost.

For every group of 3 candies:

```txt
Pay
Pay
Free
```

The free candy is automatically the cheapest among the group.

---

```txt
6,5,4,3,2,1

Pay : 6
Pay : 5
Free: 4

Pay : 3
Pay : 2
Free: 1
```

Total:

```txt
6 + 5 + 3 + 2
```

# 🛠️ Approach 2 (Your Approach)

Sort ascending.

Traverse from the end.

Maintain:

```txt
count = purchased candies
```

Pattern:

```txt
Pay
Pay
Free
Pay
Pay
Free
...
```

Whenever:

```txt
count === 2
```

skip the current candy because it becomes free.

```txt
Traverse:

6 -> Pay
```

# 🧪 Example

```txt
cost = [6,5,7,9,2,2]
```

Sort:

```txt
[2,2,5,6,7,9]
```

Process largest first:

```txt
9 -> Pay
7 -> Pay
6 -> Free

5 -> Pay
2 -> Pay
2 -> Free
```

Total:

```txt
9 + 7 + 5 + 2 = 23
```

Answer:

```txt
23
```

# ✅ Why Solution Works

To maximize discount:

```txt
Free candies should be
as expensive as possible.
```

After sorting:

```txt
largest
second largest
third largest
```

The third largest candy is the most expensive candy that can legally become free.

Therefore:

```txt
Pay
Pay
Free
```

for every group of three candies is always optimal.

---

# 💻 My Approach Code

```ts
function minimumCost(cost: number[]): number {
    cost.sort((a, b) => a - b);

    let totalCost = 0;

    for (let i = cost.length - 1; i >= 0; i -= 3) {

        totalCost += cost[i];

        if (i - 1 >= 0) {
            totalCost += cost[i - 1];
        }
    }

    return totalCost;
}
```

---

# 💻 Your Approach Code

```ts
function minimumCost(cost: number[]): number {
    cost.sort((a,b)=> a-b);

    let minCost = 0;
    let i = cost.length-1;
    let count = 0;

    while(i >= 0){ 

        // free candy
        if(count === 2){
            count = 0;
        }

        // paid candy
        else {
            minCost += cost[i];
            count++;
        }

        i--;
    }

    return minCost;
}
```

✅ Minimizing payment is often equivalent to maximizing discount.

✅ Sorting frequently exposes the optimal grouping.

✅ For every:

```txt

```


# ⚠️ Common Mistakes

### ❌ Taking arbitrary free candies

The free candy should be as expensive as possible.

---

### ❌ Ignoring sorting

Without sorting, it is difficult to maximize discount.

---

### ❌ Thinking locally

The goal is not:

```txt
Make one purchase cheaper
```

The goal is:

```txt
Maximize total discount
across all candies.
```
