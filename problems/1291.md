# 🧩 Problem #1291 - Sequential Digits

**Difficulty:** Medium     
**Topics:** `Math`     
**Link:** https://leetcode.com/problems/sequential-digits/description      

---

## 💡 Intuition

A number has **sequential digits** if every digit is exactly one greater than the previous digit.

Examples:

```text
123
4567
6789
```

At first glance, it seems reasonable to iterate through every number in the range and check whether it satisfies the property.

However, the range can be as large as

```text
10^9
```

which immediately tells us that brute force will not work.

The key realization is:

> We should generate only valid numbers instead of checking every possible number.

Taking this one step further, we notice that there are only **36 sequential-digit numbers** in the entire decimal system, making precomputation possible.

## 🔍 Key Observation

A sequential number is completely determined by two things:

* Number of digits
* Starting digit

Example

```text
digits = 4

start = 3

↓

3456
```

Even more importantly,

for every digit length there are only a few possibilities.

| Digits | Count |
| -----: | ----: |
|      2 |     8 |
|      3 |     7 |
|      4 |     6 |
|      5 |     5 |
|      6 |     4 |
|      7 |     3 |
|      8 |     2 |
|      9 |     1 |

Total

```text
8 + 7 + 6 + 5 + 4 + 3 + 2 + 1 = 36
```

So there are only **36 sequential-digit numbers** that can ever exist.

# 🧠 Attempt 1 — Brute Force (TLE)

## Idea

Iterate through every number from

```text
low
```

to

```text
high
```

and check whether it has sequential digits.

### Your Solution

```ts
function sequentialDigits(low: number, high: number): number[] {

    function isSequentialDigit(num: number): boolean {
        let _num = num;
        let previousDigit: number | null = null;

        while (_num > 0) {
            const digit = _num % 10;

            if (previousDigit !== null && digit + 1 !== previousDigit) {
                return false;
            }

            previousDigit = digit;
            _num = Math.floor(_num / 10);
        }

        return true;
    }

    const ans: number[] = [];

    for (let i = low; i <= high; i++) {
        if (isSequentialDigit(i)) {
            ans.push(i);
        }
    }

    return ans;
}
```

### Why it fails

The search space can be enormous.

### Time Complexity

```text
O((high - low) × digits)
```

Worst case

```text
≈ 10^9
```

Too slow.

# 🧠 Attempt 2 — Constructive Enumeration (Accepted)

## Idea

Instead of searching the range,

generate every possible sequential number.

A sequential number is determined by

* number of digits
* starting digit

For every valid combination,

generate the number and keep it only if it lies inside

```text
[low, high]
```

### Your Solution

```ts
function sequentialDigits(low: number, high: number): number[] {

    function generateNum(digits: number, startDigit: number): number {
        let num = startDigit;
        let lastDigit = startDigit;
        let digitCount = 1;

        while (digitCount < digits) {
            num *= 10;
            num += lastDigit + 1;
            lastDigit++;
            digitCount++;
        }

        return num;
    }

    function digitsInNum(num: number): number {
        let digitCount = 0;

        while (num > 0) {
            digitCount++;
            num = Math.floor(num / 10);
        }

        return digitCount;
    }

    const lowDigits = digitsInNum(low);
    const highDigits = digitsInNum(high);

    const ans: number[] = [];

    for (let digits = lowDigits; digits <= highDigits; digits++) {
        for (let start = 1; start <= 10 - digits; start++) {
            const num = generateNum(digits, start);

            if (num >= low && num <= high) {
                ans.push(num);
            }
        }
    }

    return ans;
}
```

### Complexity

Only **36** numbers can ever be generated.

```text
O(36)

≈ O(1)
```

# 🧠 Attempt 3 — Precompute Every Sequential Number

## Observation

Since there are only **36** valid numbers, we can precompute them once.

```ts
const sequentialNumbers = [
    12, 23, 34, 45, 56, 67, 78, 89,
    123, 234, 345, 456, 567, 678, 789,
    1234, 2345, 3456, 4567, 5678, 6789,
    12345, 23456, 34567, 45678, 56789,
    123456, 234567, 345678, 456789,
    1234567, 2345678, 3456789,
    12345678, 23456789,
    123456789
];

function sequentialDigits(low: number, high: number): number[] {
    const ans: number[] = [];

    for (const num of sequentialNumbers) {
        if (num >= low && num <= high) {
            ans.push(num);
        }
    }

    return ans;
}
```

### Complexity

```text
Time : O(36)

Space : O(36)
```

Effectively,

```text
O(1)
```

## ⚖️ Comparison

| Approach                        | Idea                                   | Time                     | Space   | Verdict    |
| ------------------------------- | -------------------------------------- | ------------------------ | ------- | ---------- |
| **1. Brute Force**              | Check every number in the range        | `O((high-low) × digits)` | `O(1)`  | ❌ TLE      |
| **2. Constructive Enumeration** | Generate only valid sequential numbers | `O(36)`                  | `O(1)`  | ✅ Optimal  |
| **3. Precomputation**           | Store all 36 sequential numbers        | `O(36)`                  | `O(36)` | ✅ Simplest |

---

## ⚖️ Why it Works

Every sequential number is uniquely determined by:

* Its length
* Its starting digit

Generating all valid combinations covers every possible sequential number exactly once.

Filtering those numbers by

```text
low <= number <= high
```

produces the required answer.

The precomputed solution is also correct because the list contains every sequential-digit number that can exist in base-10.

## 🏆 Key Takeaways

* Don't iterate over a huge search space when valid answers are rare.
* Generate only valid candidates whenever possible.
* Always ask:

```text
How many valid answers can exist?
```

Sometimes the answer is surprisingly small.

---

## 📚 Pattern Learned

### Small Candidate Space Pattern

Whenever a problem describes numbers with a strict digit pattern, ask:

```text
Can I generate every valid number?
```

If the total number of valid candidates is very small, generating or precomputing them is often simpler and faster than searching the entire range.
