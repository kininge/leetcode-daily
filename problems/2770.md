# 🧩 Problem #2770 – Maximum Number of Jumps to Reach the Last Index

**Difficulty:** Medium    
**Topics:** `Dynamic Programming`    
**Link:** https://leetcode.com/problems/maximum-number-of-jumps-to-reach-the-last-index/

---

## 📝 Problem Summary

You are given an integer array `nums` and an integer `target`.

You start at index `0` and want to reach index `n - 1`.

From index `i`, you may jump to index `j` if:

```txt
0 <= i < j < n
```
and:

```txt
-target <= nums[j] - nums[i] <= target
```

Each valid move counts as one jump.

The task is to determine the maximum number of jumps needed to reach the last index.

If it is impossible to reach the last index, return -1.

Key Observation

From every index, we may jump to multiple future indices.

This creates a directed graph: `i → j`

where an edge exists if the jump condition is valid.

Since jumps are only allowed toward the right (`i < j`), the graph is naturally a **Directed Acyclic Graph (DAG)**.

The problem therefore becomes:

```txt
Find the longest path from index 0 to index n - 1
```

This strongly suggests a Dynamic Programming solution.

### Constraints: 

- 2 <= nums.length <= 1000
- -10^9 <= nums[i] <= 10^9
- 0 <= target <= 2 * 10^9

---

## 💡 Intuition

To reach a particular index `j`, we can come from any previous index `i` such that:

```txt
-target <= nums[j] - nums[i] <= target
```

So for every index, we try all possible previous indices and choose the path that produces the maximum number of jumps.

### DP State

Let: `DP[i]`
represent: `Maximum number of jumps required to reach index i`
If an index is unreachable, its value remains: `-1`

### Transition

For every index j, check all previous indices i: `0 <= i < j`
If the jump condition is valid: `-target <= nums[j] - nums[i] <= target`
then we can extend the path: `DP[j] = max(DP[j], DP[i] + 1)`

### Why Memoization Works

The same indices may be explored multiple times from different paths.

Memoization avoids recalculating results for already-computed indices and reduces exponential recursion into polynomial time.

### Base Case

At the starting index: `DP[0] = 0`

because no jumps are needed to stay at index 0.

---

## 🛠 Approach 1 — Top-Down DP (Memoization)

### Idea

For every index `j`, try all previous indices `i` and check whether a jump from `i → j` is valid.

If valid, recursively compute the maximum jumps needed to reach `i`, then extend that path by one additional jump.

---

### Step 1 — Define DP State

We use: 
```ts
DP[i]
``` 
to store: `Maximum jumps needed to reach index i`

Possible values:
- positive integer → reachable
- `0` → starting index
- `-1` → unreachable

### Step 2 — Explore Previous Indices

To compute answer for an index: `index`
we check every: `0 <= i < index`
If: `-target <= nums[index] - nums[i] <= target`

then index `i` can jump to the current index.

### Step 3 — Build Transition

If index i is reachable: `DP[i] != -1`
then current index can extend that path: `DP[index] = max(DP[index], DP[i] + 1)`

We choose the maximum possible jumps among all valid previous indices.

### Step 4 — Use Memoization

Without memoization, the recursion would repeatedly solve the same subproblems.

Caching computed results ensures each index is solved only once.

### Why It Works

Every valid path to an index must come from a previous valid index.

By exploring all possible previous jumps and storing the best result, we guarantee that: `DP[i]`

always contains the maximum jumps needed to reach index i.

The final answer becomes: `DP[n - 1]`

---

### Code (TypeScript)

```ts
function maximumJumps(nums: number[], target: number): number {
    const n = nums.length;

    const DP = new Array(n).fill(null);

    // 0 jumps needed to stay at index 0
    DP[0] = 0;

    function DFS(index: number): number {
        if (DP[index] !== null) {
            return DP[index];
        }

        let maxJumps = -1;

        // try all previous indices
        for (let i = 0; i < index; i++) {

            const diff = nums[index] - nums[i];

            // valid jump condition
            if (-target <= diff && diff <= target) {

                const jumps = DFS(i);

                // reachable previous index
                if (jumps !== -1) {
                    maxJumps = Math.max(
                        maxJumps,
                        jumps + 1
                    );
                }
            }
        }

        DP[index] = maxJumps;

        return maxJumps;
    }

    return DFS(n - 1);
}
```

---

## ⏱ Complexity Analysis

### Time Complexity

For every index `j`, we check all previous indices: `0 <= i < j`
This results in: `O(n²)`

operations overall.

**Memoization ensures each state is computed only once.**

### Space Complexity

We use:
- a DP array of size `n`
- recursion stack depth up to `n`

So overall space complexity is: `O(n)`

#### Why It Is Efficient Enough

Constraints: `n <= 1000`

An `O(n²)` Dynamic Programming solution easily fits within limits.
