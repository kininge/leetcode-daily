# 🧩 Problem #693 – Binary Number with Alternating Bits

**Difficulty:** Easy  
**Topics:** `Bit Manipulation`, `Math`  
**Link:** https://leetcode.com/problems/binary-number-with-alternating-bits/

---

## 📝 Problem Summary

Given a positive integer `n`, determine whether its binary representation consists of alternating bits.

That means:
- No two adjacent bits should be the same.
- Valid examples:
  - `5  (101)`  → true
  - `10 (1010)` → true
- Invalid examples:
  - `7  (111)`  → false
  - `11 (1011)` → false

### Constraints
- `1 ≤ n ≤ 2^31 - 1`

---

## 🛠 Approach 1 — Direct Bit Comparison (Worst / Basic Approach)

### Idea

Traverse the binary representation bit by bit and check:

- Each bit must be different from the previous bit.
- If two adjacent bits are equal → return false.
- If we finish checking all bits → return true.

This mimics manual verification of alternating bits.

### Algorithm

1. Extract the least significant bit.
2. Shift the number right.
3. Compare the current bit with the previous one.
4. Repeat until `n` becomes 0.

### Code (TypeScript)

```ts
function hasAlternatingBits(n: number): boolean {
    let prev = n & 1;
    n >>= 1;

    while (n > 0) {
        const curr = n & 1;
        if (curr === prev) return false;
        prev = curr;
        n >>= 1;
    }

    return true;
}
```

### Complexity

- **Time**: `O(log n)` (number of bits in n)
- **Space**: `O(1)`

### Notes

- Simple and readable.
- Works for all constraints.
- Slightly procedural and not mathematically elegant.

---

## 🚀 Approach 2 — Pattern Observation Using Right Shift

### Idea

Instead of checking bits manually in a loop with previous tracking,
we can use a cleaner observation:

If bits alternate, then:

    n >> 1

should have exactly opposite bits at every position compared to `n`.

So if we XOR them:

    x = n ^ (n >> 1)

For alternating bits:
    x becomes a sequence of all 1s.

Example:

n = 5 (101)
n >> 1 = 2 (010)
XOR → 111

That means if `n` has alternating bits,
`x` must be something like:

    111...111   (all bits = 1)

---

### How do we check if a number is all 1s?

A number that is all 1s has a special property:

    x & (x + 1) === 0

Example:

x = 111 (7)
x + 1 = 1000 (8)
AND → 0000

---

### Code (TypeScript)

```ts
function hasAlternatingBits(n: number): boolean {
    const x = n ^ (n >> 1);
    return (x & (x + 1)) === 0;
}
```

### Complexity

- **Time**: `O(1)`
- **Space**: `O(1)`

### Why this is better than Approach 1

- No explicit loop
- No manual bit tracking
- Pure bit manipulation
- Cleaner and interview-impressive

This uses properties of binary patterns rather than procedural checking.

---

## 🧠 Approach 3 — Precomputed Alternating Pattern Set

### Idea

Binary numbers with alternating bits follow two possible infinite patterns:

- Starting with 1:
  1, 101, 10101, 1010101, ...
- Starting with 0:
  10, 1010, 101010, ...

Since `n ≤ 2^31 - 1`, we can precompute all valid alternating numbers within 32-bit range and store them in a Set.

Then the problem reduces to:

    return possibleNumbers.has(n);

---

### Code (TypeScript)

```ts
function hasAlternatingBits(n: number): boolean {
    const possibleNumbers = new Set([
        1,5,21,85,341,1365,5461,21845,87381,349525,
        1398101,5592405,22369621,89478485,357913941,
        1431655765,
        2,10,42,170,682,2730,10922,43690,174762,
        699050,2796202,11184810,44739242,
        178956970,715827882,2863311530
    ]);

    return possibleNumbers.has(n);
}
```

### Complexity

- **Time**: `O(1)`
- **Space**: `O(1)` (constant-sized set)

### Important Notes

- Works only because constraints are bounded to 32-bit integers.
- Not scalable if constraints change.
- Not ideal for interviews unless explicitly justified.
- Hardcoding solutions is generally discouraged.

This approach is more of a mathematical shortcut than a true algorithmic solution.

## 🔑 Final Takeaway

If bits alternate:

    n ^ (n >> 1)

produces a number of all 1s.

And a number with all 1s satisfies:

    x & (x + 1) === 0

That’s the key insight that converts the problem from procedural checking to pure bit mathematics.
