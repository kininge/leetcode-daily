# 🧩 Problem #2540 – Minimum Common Value

**Difficulty:** Easy     
**Topics:** `Array`, `Binary Search`, `Two Pointers`    
**Link:** [https://leetcode.com/problems/minimum-common-value/](https://leetcode.com/problems/minimum-common-value/)

---

## 📝 Problem Summary

You are given two integer arrays:

```txt
nums1
nums2
```

Both arrays are sorted in non-decreasing order.

Your task is to return:

```txt
smallest common integer
```

present in both arrays.

If no common value exists, return:

```txt
-1
```

---

### Example

Input:

```txt
nums1 = [1,2,3,6]
nums2 = [2,3,4,5]
```

Common elements:

```txt
2, 3
```

Minimum common value:

```txt
2
```

---

### Constraints

* `1 <= nums1.length, nums2.length <= 10^5`
* `1 <= nums1[i], nums2[j] <= 10^9`
* Both arrays are sorted in non-decreasing order

---

## 💡 Intuition

The arrays are already sorted.

This immediately suggests:

```txt
Two Pointers
```

because we can eliminate impossible candidates efficiently.

---

## 🔥 Key Observation

Suppose:

```txt
nums1[i] < nums2[j]
```

Since arrays are sorted:

```txt
nums1[i]
```

can never appear later in `nums2` before `nums2[j]`.

So:

```txt
nums1[i]
```

cannot be the minimum common value.

Therefore we safely move:

```txt
i++
```

This elimination logic is the foundation of the Two Pointer solution.

---

## 🛠 Approach 1 — Two Pointers

### Idea

Maintain two pointers:

```txt
i -> nums1
j -> nums2
```

At every step:

* if values are equal → answer found
* move pointer pointing to smaller value

Because arrays are sorted, the first common value encountered is guaranteed to be the minimum.

---

### Step-by-Step

Suppose:

```txt
nums1 = [1,2,3,6]
nums2 = [2,3,4,5]
```

Initially:

```txt
1 < 2
```

So:

```txt
1
```

cannot be common.

Move pointer in `nums1`.

Now:

```txt
2 == 2
```

Answer found.

---

### Why It Works

At every step, we eliminate one impossible candidate.

Since both arrays are sorted:

* no skipped value can become smaller common value later
* first match is guaranteed minimum

---

### Code (TypeScript)

```ts
function getCommon(nums1: number[], nums2: number[]): number {

    let i = 0;
    let j = 0;

    while (
        i < nums1.length &&
        j < nums2.length
    ) {

        if (nums1[i] === nums2[j]) {
            return nums1[i];
        }

        if (nums1[i] < nums2[j]) {
            i++;
        } else {
            j++;
        }
    }

    return -1;
}
```

## 🛠 Approach 2 — Binary Search

### Idea

Since arrays are sorted, for every element in the smaller array:

* binary search inside larger array

If found:

```txt
return current element
```

Because smaller array is traversed in sorted order, the first successful match is the minimum common value.

---

### Why It Works

Binary Search efficiently checks whether an element exists inside sorted array.

Since we process values in ascending order, the first matching value is guaranteed to be the minimum common element.

---

### Code (TypeScript)

```ts
function getCommon(nums1: number[], nums2: number[]): number {

    // always binary search in larger array
    if (nums1.length > nums2.length) {
        [nums1, nums2] = [nums2, nums1];
    }

    function binarySearch(target: number): boolean {

        let left = 0;
        let right = nums2.length - 1;

        while (left <= right) {

            const mid = (left + right) >> 1;

            if (nums2[mid] === target) {
                return true;
            }

            if (nums2[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return false;
    }

    for (const num of nums1) {

        if (binarySearch(num)) {
            return num;
        }
    }

    return -1;
}
```

## ⚖️ Approach Comparison

| Approach      | Time Complexity               | Auxiliary Space | Notes                             |
| ------------- | ----------------------------- | --------------- | --------------------------------- |
| Two Pointers  | `O(n + m)`                    | `O(1)`          | Optimal and simplest solution     |
| Binary Search | `O(min(n,m) * log(max(n,m)))` | `O(1)`          | Alternative sorted-array approach |

---

## 🏆 Which Approach Is Better?

The Two Pointer solution is preferable because:

* simpler implementation
* linear traversal
* cleaner logic
* avoids repeated Binary Searches

Since arrays are already sorted, Two Pointers naturally exploit ordering efficiently.

## 📌 Key Takeaways

* Sorted arrays often suggest:

```txt
Two Pointers
```

or:

```txt
Binary Search
```

* Two Pointer technique works by eliminating impossible candidates.

* The first common value encountered is guaranteed minimum because arrays are sorted.

* Binary Search is useful but less optimal here compared to linear merging.

## ⚠️ Common Mistakes

### 1️⃣ Using HashSet Unnecessarily

HashSet works but ignores the sorted property.

---

### 2️⃣ Forgetting Arrays Are Sorted

Sorted order enables more efficient approaches than brute force.

---

### 3️⃣ Moving Wrong Pointer

When:

```txt
nums1[i] < nums2[j]
```

only:

```txt
i++
```

is correct.

---

### 4️⃣ Returning Larger Common Value First

The first match found by Two Pointers is already the minimum common value.

No further search is required.
