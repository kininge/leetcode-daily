# 🧩 Problem #1022 – Sum of Root To Leaf Binary Numbers

**Difficulty:** Easy  
**Topics:**  `Tree`, `Depth First Search`, `Bit Manipulation`, `Recursion`  
**Link:** https://leetcode.com/problems/sum-of-root-to-leaf-binary-numbers/

---

## 📝 Problem Summary

You are given the root of a binary tree where:

- Each node contains either `0` or `1`.
- Every root-to-leaf path represents a binary number.
- The root is the **most significant bit (MSB)**.

Your task is to compute the **sum of all root-to-leaf binary numbers**.

---

### Example

Path: `0 → 1 → 1 → 0 → 1`
Binary formed: `01101`
Decimal value: `13`

You must compute this value for **every leaf path** and return their total sum.

---

### Constraints

- Number of nodes: `1 <= N <= 1000`
- `Node.val` is either `0` or `1`
- Answer fits within a 32-bit integer

---

### Key Observation

- Each path represents a binary number.
- Binary numbers are formed progressively from root to leaf.
- Instead of storing the entire path, we can build the number while traversing.

---

## 💡 Intuition

Each root-to-leaf path forms a binary number.

For example:

```txt
    1
  /  \
 0    1
```

Paths:

- `1 → 0` → binary `10` → decimal `2`
- `1 → 1` → binary `11` → decimal `3`

Total sum = `2 + 3 = 5`

---

### How are binary numbers formed?

Binary numbers are constructed like this: `previous_value * 2 + current_bit`
or equivalently: `(previous_value << 1) | current_bit`

This means:

- Moving down one level shifts bits left.
- Then we add the current node's bit.

So instead of:

- Building a string
- Converting string → decimal later

We can accumulate the numeric value directly during DFS.

---

### Key Insight

While traversing:

1. Carry the value formed so far.
2. Update it using binary rules.
3. When reaching a leaf:
   - Add the accumulated value to the total sum.

This avoids:
- String creation
- Re-parsing
- Extra memory usage

And leads to an optimal single-pass DFS solution.

---

## 🛠 Approach 1 — Build Binary Strings Then Convert (Brute Force)

### Idea

1. Perform DFS from root to leaf.
2. Build the binary path as a **string**.
3. When a leaf is reached:
   - Store the full binary string.
4. After traversal:
   - Convert each binary string to decimal.
   - Sum them.

This mirrors the literal interpretation of the problem.

---

### Why it works

- Every root-to-leaf path forms a valid binary number.
- By storing each path as a string, we preserve exact binary representation.
- Later conversion gives the correct decimal value.

The solution is straightforward but does extra work.

---

### Code (TypeScript)

```ts
function sumRootToLeaf(root: TreeNode | null): number {
    const binaryNumbers: string[] = [];

    const DFS = (node: TreeNode, carried: string) => {
        if (!node.left && !node.right) {
            binaryNumbers.push(carried + node.val);
            return;
        }

        if (node.left) DFS(node.left, carried + node.val);
        if (node.right) DFS(node.right, carried + node.val);
    };

    DFS(root!, "");

    const parseBinaryString = (num: string): number => {
        let decimal = 0;
        const n = num.length;

        for (let i = 0; i < n; i++) {
            if (num[i] === "1") {
                decimal += Math.pow(2, n - i - 1);
            }
        }

        return decimal;
    };

    let sum = 0;
    for (const binary of binaryNumbers) {
        sum += parseBinaryString(binary);
    }

    return sum;
}
```

### Complexity

- **Time Complexity:**
  - DFS traversal: `O(N)`
  - Each leaf path conversion: up to height `H`
  - Worst case: `O(N * H)` (If tree is skewed, `H = N`)
- **Space Complexity:**
  - Stores all root-to-leaf paths as strings
  - Worst case: `O(N * H)` (Plus recursion stack `O(H)`)
 
### Limitations

- Stores unnecessary intermediate strings.
- Converts binary strings to decimal separately.
- Higher memory usage.
- Not optimal for large or deep trees.

---

## 🚀 Approach 2 — Binary Accumulation During DFS (Optimal)

### Idea

Instead of:

- Building binary strings
- Converting them later

We directly construct the binary number while traversing.

At each node: `newValue = carry * 2 + node.val`
or equivalently: `newValue = (carry << 1) | node.val`

This mimics how binary numbers are formed.

When we reach a leaf:
- The accumulated value already represents the decimal number.
- Add it directly to the total sum.

---

### Why it works

Binary number formation rule:

If current number is `X`  
Appending bit `b` means: `X_new = X * 2 + b`


So during DFS:

- Each level shifts bits left
- Adds current node’s bit
- No string building
- No re-parsing

Each node is processed exactly once.

---

### Code (TypeScript)

```ts
function sumRootToLeaf(root: TreeNode | null): number {
    let sum = 0;

    const DFS = (node: TreeNode | null, carry: number) => {
        if (node === null) return;

        const newCarry = (carry << 1) | node.val;

        if (node.left === null && node.right === null) {
            sum += newCarry;
            return;
        }

        DFS(node.left, newCarry);
        DFS(node.right, newCarry);
    };

    DFS(root, 0);
    return sum;
}
```

### Complexity

- **Time Complexity:**
  - Each node visited once → `O(N)`
- **Space Complexity:**
  - Recursion stack up to tree height `H`
  - Worst case (skewed tree): `O(N)`
  - Average case (balanced tree): `O(log N)`

### Why this is better than Approach 1

- No string creation
- No separate conversion loop
- Single-pass solution
- Optimal time complexity
- Minimal memory usage

This is the expected interview solution.

---

## 📊 Comparison of Approaches

| Criteria | 🛠 Approach 1 — Build Strings | 🚀 Approach 2 — Binary Accumulation |
|-----------|--------------------------------|-------------------------------------|
| Core Idea | Store full binary path as string, convert later | Build decimal value during traversal |
| Binary Handling | Manual string parsing | Bit shifting / arithmetic accumulation |
| Time Complexity | O(N * H) | O(N) |
| Space Complexity | O(N * H) | O(H) |
| Extra Work | Converts strings after traversal | No post-processing needed |
| Memory Usage | Stores all root-to-leaf strings | Stores only recursion stack |
| Interview Strength | Basic / brute-force | Strong / optimal |
| Production Quality | Acceptable for small inputs | Preferred solution |

---

### 🏆 Final Recommendation

- Approach 1 is intuitive but inefficient due to string storage and re-parsing.
- Approach 2 is clean, optimal, and directly models binary construction.

For interviews and real-world usage,  
**Approach 2 (Binary Accumulation during DFS) is the correct choice.**
