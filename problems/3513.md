# 💡 Intuition

The problem asks for the number of **unique XOR values** obtained from

```text
nums[i] ^ nums[j] ^ nums[k]
```

where

```text
i ≤ j ≤ k
```

At first glance, this appears to be a brute-force problem because there are

```text
O(n³)
```

possible triplets.

However,

```text
n ≤ 100000
```

immediately tells us that enumerating triplets is impossible.

The key is to analyze the mathematical properties of XOR.

```text

```

```text

```

This immediately proves that every number already present in the array is also a valid XOR result.

# 🧠 Attempt 2 — Mathematical Observation (Accepted)

## Idea

Because

```text
nums
```

contains every integer from

```text
1...n
```

we only need to determine how many bits are required to represent

```text
n
```

If

```text
bits
```

bits are required,

there are

```text
2^bits
```

possible XOR values.

---

## Your Solution

```ts
function uniqueXorTriplets(nums: number[]): number {
    // edge case
    if (nums.length < 3) return nums.length;

    // because nums is a permutation of 1...n,
    // n itself is the largest number
    const largestNum = nums.length;

    let mostSegnificantBitIndex = 1;

    for (let i = 31; i >= 0; i--) {
        if (largestNum & (1 << i)) {
            mostSegnificantBitIndex = i + 1;
            break;
        }
    }

    return Math.pow(2, mostSegnificantBitIndex);
}
```

# ✅ Solution

```ts

    if (nums.length < 3) return nums.length;

    const largestNum = nums.length;
    let mostSegnificantBitIndex = 1;

    for (let i = 31; i >= 0; i--) {

            break;

```

# ⚖️ Why it Works

Since the array is a permutation of

```text
1...n
```

the only important value is the largest number,

```text
n
```

If

```text
n
```

requires

```text
bits
```

bits,

then every XOR result also requires at most

```text
bits
```

bits.

For

```text
n ≥ 3
```

every bit pattern in that range is achievable.

Therefore the total number of distinct XOR values equals

```text
2^bits
```

# 🏆 Key Takeaways

* XOR never introduces a new higher bit.
* The permutation property means only the largest value (`n`) matters.
* Count the number of representable bit patterns instead of enumerating triplets.
* Sometimes the answer depends only on the **bit-width** of the input rather than the input itself.
