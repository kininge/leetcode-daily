# 🧩 Problem #3612 – Process String with Special Operations I

**Difficulty:** Medium  
**Topics:** `String`, `Simulation`, `Array`, `Two Pointers`  
**Link:** [Process String with Special Operations I](https://leetcode.com/problems/process-string-with-special-operations-i/)

---

## 💡 Intuition

This is a pure simulation problem.

We process the string from left to right and maintain a running result.

Each character tells us exactly what to do:

```text
Letter
    ↓
Append to result

*
    ↓
Remove last character

#
    ↓
Duplicate current result

%
    ↓
Reverse current result
```

The easiest way to implement this is by using an array as a string builder.

```text
push() → append

pop() → remove last character
```

This avoids repeatedly creating new strings.

## ✅ My Solution

```ts
function processStr(s: string): string {
    const ans = [];

    // * - O(1)
    function removeLastChar(){
        if(ans.length > 0) ans.pop();
    }

    // # - O(n)
    function duplicateAns(){
        const n = ans.length;
        for(let i=0; i<n; i++) ans.push(ans[i]);
    }

    // % - O(n)
    function reverseAns(){
        let left = 0;
        let right = ans.length - 1;

        while(left < right){
            const temp = ans[left];
            ans[left] = ans[right];
            ans[right] = temp;

            left++;
            right--;
        }
    }

    for(const char of s){
        if(char === "*") removeLastChar();
        else if(char === "#") duplicateAns();
        else if(char === "%") reverseAns();
        else ans.push(char);
    }

    return ans.join('');
};
```

## 🔄 Alternative Approach

### My Approach

| Metric | Value |
| ------ | ----- |
| Time   | O(k)  |
| Space  | O(k)  |

Where:

```text
k = final result length
```

Uses an array as a string builder.

---

### Alternative

Use a normal string.

```ts
result += char;
```

and rebuild strings for every operation.

| Metric | Value                                     |
| ------ | ----------------------------------------- |
| Time   | Higher due to repeated string allocations |
| Space  | O(k)                                      |

Works fine for this problem but scales worse.

## ⚖️ Approach Comparison

| Approach                   | Time                 | Space | Notes                     |
| -------------------------- | -------------------- | ----- | ------------------------- |
| String Builder (Array)     | O(k)                 | O(k)  | Recommended               |
| Direct String Manipulation | O(k²) in worst cases | O(k)  | Simpler but less scalable |

## ⏱️ Complexity Analysis

Let:



Time:


```text

```

Time:


```text

```

---

```text
O(k)
```

Because:

```text
s.length ≤ 20
```

```text
Time Complexity: O(k)
```

```text
k = final result length
```

## 🏆 Key Takeaways

✅ When repeatedly modifying strings:

```text
Use Array
      ↓
Join Once
```

instead of continuously creating new strings.

---

✅ Simulation problems are often easier than they first appear.

Just carefully follow the instructions.

---

✅ Reverse operations often naturally lead to:

```text
Two Pointers

left ↔ right
```

---

✅ Duplication can be done efficiently by iterating only over the original length.

```ts
const n = ans.length;
```

This prevents newly added characters from being duplicated again.
