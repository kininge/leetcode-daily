# 🧩 Problem #3754 - Concatenate Non-Zero Digits and Multiply by Sum I

## 💡 Intuition

We need to create a new number by removing all `0` digits while preserving the order of the remaining digits.

At the same time, we also need the sum of those non-zero digits.

A straightforward approach is:

1. Traverse each digit of `n`.
2. Collect every non-zero digit.
3. Compute their sum.
4. Reconstruct the new number.
5. Return:

```text
newNumber × sum
```

An optimization is that if the original number contains **no zero digits**, then the reconstructed number is exactly the original number, so reconstruction can be skipped.

## 🔍 Key Observation

Digits are extracted from an integer **right to left**.

To preserve their original order, we store them first and reconstruct the number in reverse order.

Additionally,

```text
If there are no zero digits,

newNumber == original number
```

so reconstruction can be skipped.

## ✅ Solution

```ts
function sumAndMultiply(n: number): number {
    const x = [];
    let sum = 0;
    let num = n;
    let countOfZeros = 0;

    while (num > 0) {
        const reminder = num % 10;
        num = Math.floor(num / 10);

        if (reminder > 0) {
            sum += reminder;
            x.push(reminder);
        } else {
            countOfZeros++;
        }
    }

    // No zero digits, so reconstruction is unnecessary.
    if (countOfZeros === 0) {
        return n * sum;
    }

    let newNum = 0;

    for (let i = x.length - 1; i >= 0; i--) {
        newNum *= 10;
        newNum += x[i];
    }

    return newNum * sum;
}
```

---

## ⚖️ Why it Works

During traversal:

* Every non-zero digit is added exactly once to `sum`.
* Every non-zero digit is stored in the order it is encountered (right to left).

Reconstructing the stored digits in reverse order restores the original left-to-right order, producing exactly the concatenation of all non-zero digits.

If no zero digits exist, the reconstructed number is identical to the original input, so returning `n * sum` is correct.

Thus, the algorithm always computes:

```text
x × (sum of digits in x)
```

which is exactly the required result.

## 🏆 Key Takeaways

* Integer digits are naturally extracted from **right to left**.
* Preserving digit order often requires temporary storage or reconstruction.
* Small observations (like skipping reconstruction when no zeros exist) can simplify work without changing asymptotic complexity.
* Digit manipulation problems are often clean simulation problems rather than algorithmically complex ones.
