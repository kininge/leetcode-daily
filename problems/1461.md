# 🧩 Problem #1461 – Check If a String Contains All Binary Codes of Size K

**Difficulty:** Medium  
**Topics:**  `String`, `Sliding Window`, `Bit Manipulation`, `Hash Map`  
**Link:** https://leetcode.com/problems/check-if-a-string-contains-all-binary-codes-of-size-k/

---

## 📝 Problem Summary

Given a binary string `s` and an integer `k`, determine whether **every possible binary code of length `k`** exists as a substring of `s`.

A binary code of size `k` means:
- A string of length `k`
- Each character is either `'0'` or `'1'`
- Total possible distinct codes = `2^k`

Return:

- `true` → if all `2^k` possible binary strings appear in `s`
- `false` → otherwise

---

### Example 1 

```txt
Input: s = "00110110", k = 2
Output: true
```
All possible binary codes of length 2: 00, 01, 10, 11
Each appears at least once in `s`.

### Example 2

```txt
Input: s = "0110", k = 2
Output: false
```
Possible codes: 00, 01, 10, 11
Substring `"00"` does not exist.

---

### Constraints

- `1 <= s.length <= 5 * 10^5`
- `s[i]` is `'0'` or `'1'`
- `1 <= k <= 20`

---

### Key Observation

- Total possible codes = `2^k`
- We only need to check substrings of size `k`
- If we find exactly `2^k` distinct substrings of size `k`,
  then all binary codes are present.

---

## 💡 Intuition

We are asked to verify whether **all possible binary strings of length `k`**
exist inside `s`.

First, understand the scale:

- Total possible binary codes of length `k` = `2^k`
- If `k = 3`, total codes = 8
- If `k = 20`, total codes = 1,048,576

So instead of generating all possible codes and checking each one (expensive),
we reverse the thinking:

> Slide a window of size `k` over `s` and collect all distinct substrings.

If the number of distinct substrings of size `k` equals `2^k`,
then every possible code must exist.

---

### Important Constraint Insight

Before even processing:

If: 
s.length < k
→ Impossible.

So if:
s.length - k + 1 < 2^k
→ Even theoretically impossible to contain all codes.

This gives an early optimization opportunity.

---

### Core Strategy

1. Use a sliding window of length `k`.
2. Track all distinct windows.
3. If distinct count equals `2^k` → return `true`.
4. Otherwise → `false`.

The only real question is:
How efficiently do we track these substrings?

Also, maximum substrings of size `k` in `s`: s.length - k + 1

---

## 🛠 Approach 1 — HashSet + Substring (Direct Sliding Window)

### Idea

Slide a window of size `k` across the string and:

1. Extract each substring of length `k`
2. Insert it into a `Set`
3. At the end, check if the number of distinct substrings equals `2^k`

If yes → all binary codes exist.  
If no → at least one code is missing.

---

### Why it works

- Total possible binary codes of length `k` = `2^k`
- Each window of length `k` gives one candidate code
- If we collect all distinct windows and the count matches `2^k`,
  then every possible binary string must have appeared.

This reverses the brute-force thinking:
Instead of generating all `2^k` possibilities,
we validate them by counting distinct substrings.

---

### Code (TypeScript)

```ts
function hasAllCodes(s: string, k: number): boolean {
    const numbers = new Set<string>();

    for (let i = 0; i + k <= s.length; i++) {
        const subset = s.substring(i, i + k);
        numbers.add(subset);
    }

    return numbers.size === Math.pow(2, k);
}
```

---

### Complexity

- **Time Complexity:**
  - Sliding window runs `O(n)` times
  - substring(i, i + k) takes O(k)
  - Total: `O(n * k)`

Since k ≤ 20, this behaves close to linear in practice.

- **Space Complexity:**
  - Up to `2^k` substrings stored
  - Each substring has length `k`

Worst-case space: `O(2^k * k)`
For `k = 20`, this can exceed one million stored strings.

---

### Limitations

- Creates a new substring on every iteration
- Stores full strings instead of compact representation
- Higher memory usage
- Slightly heavier constant factor

---

## 🚀 Approach 2 — Rolling Bitmask (Improved / Memory Efficient)

### Idea

Instead of:

- Creating substrings of length `k`
- Storing full strings in a `Set`

We treat each window as a **binary number**.

Example:
```txt
"101" → 5
"011" → 3
```

We maintain a rolling integer value using bit operations:

For each new character:
1. Left shift previous value.
2. Add the new bit.
3. Remove the leftmost bit using a mask.

This avoids substring creation entirely.

---

### Why it works

A window of size `k` can be represented as an integer between: `0 to (2^k - 1)`

Instead of storing strings, we:

- Maintain a boolean array of size `2^k`
- Mark each encountered pattern as seen

Once we mark `2^k` distinct values → return `true`.

This reduces:

- Memory overhead
- String allocations
- Constant factors

---

### Code (TypeScript)

```ts
function hasAllCodes(s: string, k: number): boolean {
    const total = 1 << k;
    const seen = new Array<boolean>(total).fill(false);

    let mask = total - 1;
    let window = 0;
    let count = 0;

    for (let i = 0; i < s.length; i++) {
        window = ((window << 1) & mask) | (s[i] === '1' ? 1 : 0);

        if (i >= k - 1) {
            if (!seen[window]) {
                seen[window] = true;
                count++;
                if (count === total) return true;
            }
        }
    }

    return false;
}
```
---

### Complexity

- **Time Complexity:**
  - Single pass through string
  - Bit operations are `O(1)`

Total: `O(n)`

- **Space Complexity:**
  - Boolean array of size `2^k`

Total: `O(2^k)`

Note: This is the minimum possible because there are `2^k` possible codes.

---

### Why this is better than Approach 1

- Avoids `substring` creation
- Avoids storing large strings
- Uses compact integer representation
- Faster in practice
- Cleaner sliding window abstraction

This is the preferred solution in performance-sensitive interviews.

---

## 📊 Comparison of Approaches

| Criteria | 🛠 Approach 1 — HashSet + Substring | 🚀 Approach 2 — Rolling Bitmask |
|-----------|--------------------------------------|----------------------------------|
| Core Idea | Store all substrings of size `k` in a Set | Represent window as integer using bitmask |
| Window Update | Create new substring each time | Bit shift + mask in O(1) |
| Time Complexity | O(n * k) | O(n) |
| Space Complexity | O(2^k * k) | O(2^k) |
| Memory Usage | Stores full strings | Stores compact boolean flags |
| Constant Factor | Higher (string allocation) | Lower (bit operations) |
| Interview Strength | Good first solution | Strong optimized solution |
| Production Efficiency | Acceptable | Preferred |

---

### 🏆 Final Recommendation

- Approach 1 is clean and intuitive.
- Approach 2 is more memory-efficient and avoids substring creation.
- Both are asymptotically optimal in terms of time.

For interviews and performance-sensitive scenarios,  
**Approach 2 (Rolling Bitmask) is the preferred solution.**
