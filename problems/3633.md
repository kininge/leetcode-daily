# 🧩 Problem #3633 – Earliest Finish Time for Land and Water Rides I

**Difficulty:** Easy     
**Topics:** `Simulation`      
**Link:** https://leetcode.com/problems/earliest-finish-time-for-land-and-water-rides-i/    

---

# 📖 Problem Summary

A tourist must take:

```txt
1 Land Ride
+
1 Water Ride
```

The rides can be taken in either order:

```txt
Land → Water
```

or

```txt
Water → Land
```

A ride can only start after:

* it opens
* the previous ride finishes

Return the earliest possible finishing time.

---

# 💡 Intuition

For a chosen pair:

```txt
Land Ride i
Water Ride j
```

there are only two possible plans:

```txt
Land → Water
```

or

```txt
Water → Land
```

Since:

```txt
n <= 100
m <= 100
```

we can simply:

```txt
Try every pair
Try both orders
Take the minimum finish time
```

---

# 🔍 Key Observation

Suppose:

```txt
Land finishes at time = landEndTime
```

Can we immediately start the water ride?

Two possibilities:

### Water ride already open

```txt
waterStartTime <= landEndTime
```

Start immediately.

---

### Water ride not open yet

```txt
waterStartTime > landEndTime
```

Wait until it opens.

---

Both cases can be merged into:

```ts
Math.max(
    landEndTime,
    waterStartTime
)
```

This makes the implementation much cleaner.

---

# 🛠️ Approach

For every:

```txt
Land Ride i
```

and

```txt
Water Ride j
```

calculate:

### Land → Water

```ts
landEndTime =
    landStartTime[i]
    + landDuration[i]

landThenWater =
    Math.max(
        landEndTime,
        waterStartTime[j]
    )
    + waterDuration[j]
```

---

### Water → Land

```ts
waterEndTime =
    waterStartTime[j]
    + waterDuration[j]

waterThenLand =
    Math.max(
        waterEndTime,
        landStartTime[i]
    )
    + landDuration[i]
```

Keep the smallest answer among all combinations.

---

# 🧪 Example

```txt
landStartTime = [2,8]
landDuration  = [4,1]

waterStartTime = [6]
waterDuration  = [3]
```

---

### Land Ride 0 → Water Ride 0

Land:

```txt
Start = 2
End   = 6
```

Water:

```txt
Start = max(6, 6)
      = 6

Finish = 9
```

---

### Water Ride 0 → Land Ride 1

Water:

```txt
Start = 6
End   = 9
```

Land:

```txt
Start = max(9, 8)
      = 9

Finish = 10
```

Minimum:

```txt
9
```

# ✅ Why Solution Works

For every possible pair:

```txt
Land Ride i
Water Ride j
```

we evaluate:

```txt
Land → Water
```

and

```txt
Water → Land
```

These are the only valid orders.

Since we examine:

```txt
all pairs
+
both orders
```

the minimum finish time found is guaranteed to be the optimal answer.

---

# 💻 Code

```ts
function earliestFinishTime(
    landStartTime: number[],
    landDuration: number[],
    waterStartTime: number[],
    waterDuration: number[]
): number {

    let smallestEndTime = 10000;

    for (let i = 0; i < landStartTime.length; i++) {

        for (let j = 0; j < waterStartTime.length; j++) {

            const landEndTime =
                landStartTime[i]
                + landDuration[i];

            const waterEndTime =
                waterStartTime[j]
                + waterDuration[j];

            const landThenWater =
                Math.max(
                    landEndTime,
                    waterStartTime[j]
                )
                + waterDuration[j];

            const waterThenLand =
                Math.max(
                    waterEndTime,
                    landStartTime[i]
                )
                + landDuration[i];

            smallestEndTime = Math.min(
                smallestEndTime,
                landThenWater,
                waterThenLand
            );
        }
    }

    return smallestEndTime;
}
```

# ⚠️ Common Mistakes

### ❌ Forgetting to try both orders

Need to check:

```txt
Land → Water
```

and

```txt
Water → Land
```

---

### ❌ Ignoring waiting time

If a ride is not open yet:

```txt
Finish first ride
        ↓
Wait
        ↓
Start second ride
```

must be handled.

---

### ❌ Using complicated optimizations

Constraints are small enough that:

```txt
O(n × m)
```

is already optimal and easy to implement.
