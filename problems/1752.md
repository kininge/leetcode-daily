# 🧩 Problem #1752 – Check if Array Is Sorted and Rotated

**Difficulty:** Easy    
**Topics:** `Array`    
**Link:** [https://leetcode.com/problems/check-if-array-is-sorted-and-rotated/](https://leetcode.com/problems/check-if-array-is-sorted-and-rotated/)   

---

## 📝 Problem Summary

You are given an integer array:

```txt
nums
```

Determine whether the array was originally sorted in non-decreasing order and then rotated some number of times.

Rotation may be:

```txt
0 times
```

which means the array can already be sorted.

Duplicates are allowed.

---

### Example

Valid rotated sorted array:

```txt
[3,4,5,1,2]
```

Original sorted array:

```txt
[1,2,3,4,5]
```

Rotated by:

```txt
2 positions
```

---

### Constraints

* `1 <= nums.length <= 100`
* `1 <= nums[i] <= 100`

---

## 💡 Intuition

A sorted rotated array contains:

```txt
at most ONE breaking point
```

Example:

```txt
[3,4,5,1,2]
```

Only:

```txt
5 > 1
```

breaks sorted order.

If multiple decreasing points exist:

```txt
array cannot come from single rotation
```

---

## 🔥 Key Observation

For a valid sorted rotated array:

```txt
ordering should fail at most once
```

That failure represents:

```txt
rotation boundary
```

Example:

```txt
[3,4,5,1,2]
```

Boundary:

```txt
5 -> 1
```

---

## 🛠 Approach 1 — Count Dips (Your Approach)

### Idea

Traverse the array and count how many times:

```txt
nums[i-1] > nums[i]
```

occurs.

This represents a:

```txt
sorted-order violation
```

A valid rotated sorted array can have:

```txt
at most one dip
```

---

### Important Extra Validation

Suppose:

```txt
[2,1,3,4]
```

This has only one dip:

```txt
2 > 1
```

but it is NOT a valid rotated sorted array.

Why?

Because after rotation:

```txt
first partition must be >= second partition
```

Here:

```txt
2 < 4
```

which breaks rotation ordering.

---

### Why It Works

A rotated sorted array consists of:

```txt
TWO sorted partitions
```

Example:

```txt
[3,4,5] + [1,2]
```

Only one transition point between partitions may break ordering.

Additional breaks invalidate the structure.

---

### Code (TypeScript)

```ts
function check(nums: number[]): boolean {

    const n = nums.length;

    // edge case
    if (n === 1) {
        return true;
    }

    let noOfDips = 0;

    for (let i = 1; i < n; i++) {

        if (nums[i - 1] > nums[i]) {
            noOfDips++;
        }

        // only one dip allowed
        if (noOfDips > 1) {
            return false;
        }
    }

    // validate rotation boundary
    if (
        noOfDips === 1 &&
        nums[0] < nums[n - 1]
    ) {
        return false;
    }

    return true;
}
```

## 🛠 Approach 2 — Circular Ordering Check (Elegant)

### Core Idea

Instead of separately handling:

* internal ordering
* rotation boundary

we can treat array as:

```txt
circular
```

and count how many times:

```txt
nums[i] > nums[(i+1)%n]
```

occurs.

---

### Why Circular Traversal Works

A rotated sorted array remains sorted:

```txt
except at exactly one boundary
```

Example:

```txt
[3,4,5,1,2]
```

Violations:

```txt
5 > 1
```

Only one violation exists.

---

### Invalid Example

```txt
[2,1,3,4]
```

Violations:

```txt
2 > 1
4 > 2
```

Two violations.

Therefore:

```txt
invalid
```

---

### Why It Works

Circular traversal naturally includes:

```txt
last element -> first element
```

which automatically validates the rotation boundary.

This removes the need for separate edge-case checks.

---

### Code (TypeScript)

```ts
function check(nums: number[]): boolean {

    let dips = 0;

    const n = nums.length;

    for (let i = 0; i < n; i++) {

        if (
            nums[i] >
            nums[(i + 1) % n]
        ) {
            dips++;
        }
    }

    return dips <= 1;
}
```

## ⚖️ Approach Comparison

| Approach       | Main Idea                          | Notes                    |
| -------------- | ---------------------------------- | ------------------------ |
| Count Dips     | Count internal ordering violations | More intuitive initially |
| Circular Check | Count circular ordering breaks     | Cleaner and more elegant |

---

## 🏆 Which Approach Is Better?

The Circular Check approach is usually preferred because:

* shorter logic
* naturally handles rotation boundary
* fewer edge cases
* mathematically elegant

The dip-counting solution is still excellent for understanding the problem structure.

| Operation       | Complexity |
| --------------- | ---------- |
| Traversal       | `O(n)`     |
| Auxiliary Space | `O(1)`     |

## 📌 Key Takeaways

* Rotated sorted arrays contain at most one ordering violation.

* Circular traversal often simplifies rotation problems.

* Rotation boundary is simply:

```txt
largest element -> smallest element
```

* Many rotation problems become easier when viewed as circular arrays.

---

## ⚠️ Common Mistakes

### 1️⃣ Only Counting Internal Dips

Without validating:

```txt
last element -> first element
```

some invalid arrays may incorrectly pass.

---

### 2️⃣ Forgetting Zero Rotation Case

Already sorted arrays are valid.

---

### 3️⃣ Assuming Strictly Increasing Order

Duplicates are allowed.

So:

```txt
1,1,2,2
```

is valid.

---

### 4️⃣ Overcomplicating Rotation Logic

Rotation problems often become simpler with:

```txt
modulo indexing
```

and circular traversal.
