# 🧩 Problem #1833 – Maximum Ice Cream Bars

**Difficulty:** Medium    
**Topics:** `Greedy`, `Sorting`    
**Link:** [Maximum Ice Cream Bars](https://leetcode.com/problems/maximum-ice-cream-bars/)     

---

## 💡 Intuition

Every ice cream bar contributes exactly:

```text
+1 ice cream
```

There is no separate value/profit array.

This means:

```text
More Expensive Ice Cream
        ≠
More Benefit
```

Every ice cream is equally valuable.

So if our goal is:

```text
Maximize Number Of Ice Creams
```

the optimal strategy is:

```text
Buy Cheapest First
```

because cheaper ice creams leave more money for future purchases.

### Initial Thought

```text

    +

```

```text

```

---

### Observation

In Knapsack:

```text
Weight
+

```

exist separately.

Here:

```text
Every Ice Cream
        ↓
Value = 1
```

So DP is unnecessary.

---

### Better Idea

```text
Sort Costs
      ↓
Buy Cheapest First
      ↓
Stop When Money Runs Out
```

---

### Intended Optimization

Since:

```text
1 <= costs[i] <= 100000
```

we can also use:

```text
Counting Sort
```

instead of comparison sorting.

## ✅ My Solution

```ts
// it look like 1-0 Knap-sack DP but there is no value array in this
// we can solve this byr sort and gready way
function maxIceCream(costs: number[], coins: number): number {
    costs.sort((a, b)=> a-b);

    let count = 0;
    let money = coins;

    for(const cost of costs){
        if(money >= cost) {
            count++;
            money -= cost;
        }else{
            break;
        }
    }

    return count;
};
```

## 🔄 Alternative Approach

### My Approach (Sorting + Greedy)

| Metric | Value            |
| ------ | ---------------- |
| Time   | O(n log n)       |
| Space  | O(1) or O(log n) |

Sort and purchase from cheapest to most expensive.

---

### Alternative (Counting Sort)

Since:

```text
costs[i] <= 100000
```

we can build:

```text
frequency[cost]
```

Example:

```text
costs = [1,3,2,4,1]

frequency:

1 → 2
2 → 1
3 → 1
4 → 1
```

Then process costs from:

```text
1
↓
2
↓
3
↓
...
```

without sorting.

---

## ⚖️ Approach Comparison

| Approach               | Time       | Space | Notes             |
| ---------------------- | ---------- | ----- | ----------------- |
| Sorting + Greedy       | O(n log n) | O(1)  | Easier            |
| Counting Sort + Greedy | O(n + k)   | O(k)  | Intended Solution |

Where:

```text
k = 100000
```
