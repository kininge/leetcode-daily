# 🧩 Problem #3300 – Minimum Element After Replacement With Digit Sum

**Difficulty:** Easy     
**Topics:** `Math`, `Array`    
**Link:** https://leetcode.com/problems/minimum-element-after-replacement-with-digit-sum/

---

# 📖 Problem Summary

Replace every number with the sum of its digits.

Return the minimum value after all replacements.

---

# 💡 Intuition

The problem says:

```txt
Replace every element
Then find minimum
```

But we don't actually need to modify the array.

We only care about:

```txt
Minimum digit sum
```

So for each number:

```txt
Compute digit sum
Update minimum
```

---

# 🔍 Key Observation

For a number:

```txt
123
```

Digit sum is:

```txt
1 + 2 + 3 = 6
```

We can extract digits using:

```txt
num % 10
```

and remove the last digit using:

```txt
Math.floor(num / 10)
```

---

# 🛠️ Approach

For every number:

1. Calculate its digit sum.
2. Keep track of the minimum digit sum seen so far.
3. Return the minimum.

### Visual

```txt
nums = [10,12,13,14]

10 → 1
12 → 3
13 → 4
14 → 5

Minimum = 1
```

---

# 🧪 Step-by-step Example

```txt
nums = [999,19,199]
```

Digit sums:

```txt
999 → 9 + 9 + 9 = 27

19 → 1 + 9 = 10

199 → 1 + 9 + 9 = 19
```

Result:

```txt
min(27, 10, 19) = 10
```

---

# ✅ Why Solution Works

For every element:

```txt
original value
        ↓
digit sum
```

Since we compute the digit sum of every number exactly once and continuously track the smallest value, the final answer is guaranteed to be the minimum element after replacement.

---

# 💻 Code

```ts
function minElement(nums: number[]): number {

    function digitSum(num: number): number {
        let _num = num;
        let sum = 0;

        while (_num > 0) {
            sum += _num % 10;
            _num = Math.floor(_num / 10);
        }

        return sum;
    }

    let minSum = 36; // 9999 -> 9+9+9+9 = 36

    for (const num of nums) {
        minSum = Math.min(minSum, digitSum(num));
    }

    return minSum;
}
```

---

# ⏱️ Complexity Analysis

| Operation | Complexity |
| --------- | ---------- |
| Time      | `O(n)`     |
| Space     | `O(1)`     |

Since every number contains at most 4 digits (`nums[i] <= 10^4`), digit sum computation is effectively constant time.

# 🧠 Key Takeaways

✅ We don't need to actually replace elements in the array.

✅ Transform + aggregate is a common pattern.

```txt
Apply function
        +
Track minimum
```

✅ Digit extraction pattern:

```ts
digit = num % 10;
num = Math.floor(num / 10);
```

is a very common technique in math problems.

# ⚠️ Common Mistakes

### ❌ Creating a new transformed array

```txt
Compute digit sums
Store all values
Find minimum later
```

This uses unnecessary extra space.

---

### ❌ Forgetting to remove digits

After using:

```ts
num % 10
```

don't forget:

```ts
num = Math.floor(num / 10);
```

otherwise the loop never progresses.
