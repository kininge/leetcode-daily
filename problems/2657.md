# 🧩 Problem #2657 – Find the Prefix Common Array of Two Arrays

**Difficulty:** Medium     
**Topics:** `Array`, `Hash Map`, `Prefix Sum`     
**Link:** [https://leetcode.com/problems/find-the-prefix-common-array-of-two-arrays/](https://leetcode.com/problems/find-the-prefix-common-array-of-two-arrays/)

---

## 📝 Problem Summary

You are given two integer permutations:

```txt
A
B
```

Both arrays have length:

```txt
n
```

For every index `i`, define:

```txt
C[i]
```

as:

```txt
count of numbers present in both prefixes
A[0..i] and B[0..i]
```

Return the prefix common array:

```txt
C
```

---

### Example

Input:

```txt
A = [1,3,2,4]
B = [3,1,2,4]
```

Output:

```txt
[0,2,3,4]
```

Explanation:

| Index | Prefix A  | Prefix B  | Common Numbers | Count |
| ----- | --------- | --------- | -------------- | ----- |
| 0     | [1]       | [3]       | none           | 0     |
| 1     | [1,3]     | [3,1]     | 1,3            | 2     |
| 2     | [1,3,2]   | [3,1,2]   | 1,2,3          | 3     |
| 3     | [1,3,2,4] | [3,1,2,4] | 1,2,3,4        | 4     |

---

### Constraints

* `1 <= n <= 50`
* `1 <= A[i], B[i] <= n`
* `A` and `B` are permutations

---

## 💡 Intuition

A brute-force approach would:

* compute prefix intersections repeatedly
* recalculate common elements from scratch

This would be unnecessary.

---

## 🔥 Key Observation

At every index:

```txt
Only TWO new elements are introduced
```

* `A[i]`
* `B[i]`

So instead of recomputing intersections:

```txt
incrementally update common count
```

This turns the problem into a running-state tracking problem.

---

## 🛠 Approach 1 — Two Sets + Incremental Counting

### Idea

Maintain:

* `seenA` → elements seen in `A`
* `seenB` → elements seen in `B`

At every index:

* add current elements into sets
* check whether newly added elements already exist in opposite set
* update common count accordingly

---

### Step-by-Step

Suppose:

```txt
A = [1,3,2,4]
B = [3,1,2,4]
```

At:

```txt
i = 1
```

We add:

```txt
3 into seenA
1 into seenB
```

Now:

```txt
3 exists in seenB
1 exists in seenA
```

So:

```txt
common count += 2
```

---

## ⚠️ Double Counting Case

Suppose:

```txt
A[i] === B[i]
```

Then both checks become true.

Example:

```txt
A[i] = 4
B[i] = 4
```

Without correction:

```txt
same value counted twice
```

So we subtract:

```ts
commonNums--
```

This fixes duplicate counting.

---

### Why It Works

Every number becomes common exactly when:

```txt
it has appeared in both arrays
```

The algorithm tracks precisely when that happens.

Since only two new elements are introduced per step, updates remain efficient.

---

### Code (TypeScript)

```ts
function findThePrefixCommonArray(
    A: number[],
    B: number[]
): number[] {

    let commonNums = 0;

    const seenA = new Set<number>();
    const seenB = new Set<number>();

    const ans: number[] = [];

    for (let i = 0; i < A.length; i++) {

        seenA.add(A[i]);
        seenB.add(B[i]);

        if (seenB.has(A[i])) {
            commonNums++;
        }

        if (seenA.has(B[i])) {
            commonNums++;
        }

        // prevent double counting
        if (A[i] === B[i]) {
            commonNums--;
        }

        ans.push(commonNums);
    }

    return ans;
}
```

## 🛠 Approach 2 — Frequency Counting

### Idea

Instead of maintaining two separate sets:

Maintain a frequency array:

```txt
freq[x]
```

representing:

```txt
how many times x has appeared
across both arrays
```

Whenever frequency becomes:

```txt
2
```

that number has now appeared in both arrays.

So:

```txt
common count increases
```

---

### Why It Works

Because:

* arrays are permutations
* each value appears exactly once in each array

So frequency reaching:

```txt
2
```

means:

```txt
value has appeared in both prefixes
```

---

### Step-by-Step

Suppose:

```txt
A = [1,3,2,4]
B = [3,1,2,4]
```

At:

```txt
i = 1
```

We process:

```txt
3 from A
1 from B
```

Both frequencies become:

```txt
2
```

So:

```txt
common count += 2
```

---

### Code (TypeScript)

```ts
function findThePrefixCommonArray(
    A: number[],
    B: number[]
): number[] {

    const n = A.length;

    const freq = new Array(n + 1).fill(0);

    const ans: number[] = [];

    let common = 0;

    for (let i = 0; i < n; i++) {

        freq[A[i]]++;

        if (freq[A[i]] === 2) {
            common++;
        }

        freq[B[i]]++;

        if (freq[B[i]] === 2) {
            common++;
        }

        ans.push(common);
    }

    return ans;
}
```

## ⚖️ Approach Comparison

| Approach           | Time Complexity | Auxiliary Space | Notes                       |
| ------------------ | --------------- | --------------- | --------------------------- |
| Two Sets           | `O(n)`          | `O(n)`          | Very intuitive and explicit |
| Frequency Counting | `O(n)`          | `O(n)`          | More compact and elegant    |

---

## 🏆 Which Approach Is Better?

Both approaches are optimal.

However:

* Set-based solution is easier to understand initially
* Frequency approach is slightly cleaner and more elegant

Because constraints are small:

```txt
n <= 50
```

both are perfectly acceptable.

## 📌 Key Takeaways

* Prefix problems often benefit from:

```txt
incremental updates
```

instead of recomputation.

* Sets are useful for tracking prefix membership.

* Frequency counting can simplify intersection-style problems.

* Double-counting edge cases are common when processing two streams simultaneously.

---

## ⚠️ Common Mistakes

### 1️⃣ Recomputing Prefix Intersections Repeatedly

This leads to unnecessary repeated work.

---

### 2️⃣ Forgetting Double Counting Case

When:

```txt
A[i] === B[i]
```

same value may accidentally be counted twice.

---

### 3️⃣ Ignoring Permutation Property

Permutation guarantees uniqueness within each array.

This simplifies frequency logic significantly.

---

### 4️⃣ Using Nested Loops

Brute-force prefix comparisons increase complexity unnecessarily.
