# 🧩 Problem #3093 – Longest Common Suffix Queries

**Difficulty:** Hard    
**Topics:** `Trie`, `String`, `Hash Map`      
**Link:** [Leetcode](https://leetcode.com/problems/longest-common-suffix-queries/)    

---

# 1. Problem Summary

For every query word:

- find the word from `wordsContainer`
- having the **longest common suffix**

Tie breaking:
1. Smaller length wins
2. If same length → earlier index wins

Return the selected word index for every query.

---

# 2. Intuition

This is a suffix matching problem.

Classic transformation:

```txt
suffix matching
→ reverse strings
→ prefix matching
```

Example:

```txt
"abcd"
```

Reverse:

```txt
"dcba"
```

Now suffix matching becomes prefix traversal.

This strongly suggests:

```txt
Trie
```

---

# 3. Key Observation

A trie node can represent:

```txt
a suffix group
```

Example:

```txt
root
 └── d
      └── c
           └── b
```

represents suffixes:

```txt
"d"
"cd"
"bcd"
```

The important optimization:

```txt
Trie node should already know
the best answer for this suffix.
```

instead of:
- finding all candidate words later
- rescanning subtree repeatedly

---

# 4. First Trie Approach (Failed)

## Initial Thinking

Build:
- reversed trie

For query:
- traverse matching suffix
- BFS subtree
- collect candidate words
- choose:
  - shortest length
  - earliest index

---

## Why This Gives TLE

Suppose queries are:

```txt
"gh"
"egh"
"zzgh"
```

All may land near same subtree.

But BFS repeatedly scans:
- same descendants
- same words
- same comparisons

again and again.

---

## Repeated Work

```txt
query
  → suffix node
      → BFS subtree
```

Trie helps matching,
but subtree rescanning still remains expensive.

---

# 5. Second Approach – HashMap Aggregation

## Important Insight

Instead of computing answers during query:

```txt
Precompute:
suffix -> best answer
```

Example:

```txt
"d"     -> best index
"cd"    -> best index
"bcd"   -> best index
"abcd"  -> best index
```

This removes:
- BFS
- subtree traversal
- repeated query computation

---

# 6. Why HashMap Approach Still Gives TLE

The hidden bottleneck was:

```ts
suffix += word[i];
```

---

## Hidden Complexity

Strings are immutable.

So:

```txt
"h"
"hg"
"hgf"
"hgfe"
```

creates NEW strings every iteration.

---

## Actual Complexity

For word length `n`:

```txt
1 + 2 + 3 + ... + n
= O(n²)
```

PER WORD.

---

## Why It Becomes Slow

Constraints:

```txt
Total characters <= 5 * 10^5
```

Repeated string creation becomes extremely expensive.

Even though:

```txt
HashMap lookup = O(1)
```

string construction itself becomes the bottleneck.

---

# 7. Final Optimized Insight

Trie nodes themselves already represent suffixes.

So instead of:

```txt
Map<suffixString, bestAnswer>
```

store directly on trie nodes:

```txt
node.ans
```

Now:
- no suffix string creation
- no hashing long strings
- no subtree traversal

---

# 8. Final Approach

Each trie node stores:

| Property | Meaning |
|---|---|
| `children` | Next characters |
| `ans` | Best word index for this suffix |

---

## During Insertion

Traverse reversed word.

At EVERY node:
- update best answer if:
  - current word is shorter
  - OR same length but earlier index

---

## During Query

Traverse reversed query.

As long as path exists:
- continue traversal
- current node already stores optimal answer

No BFS needed.

---

# 9. Step-by-step Example

## wordsContainer

```txt
["abcd", "bcd", "xbcd"]
```

Reversed insertion:

```txt
"dcba"
"dcb"
"dcbx"
```

Trie:

```txt
root
 └── d
      └── c
           └── b
```

Node `"d->c->b"` represents suffix:

```txt
"bcd"
```

Best answer stored there:

```txt
index = 1
```

because:

```txt
"bcd"
```

has smallest length.

---

## Query

```txt
"zzbcd"
```

Reverse:

```txt
"dcbzz"
```

Traversal:

```txt
d -> c -> b
```

Longest matched suffix:

```txt
"bcd"
```

Answer is already stored on node.

---

# 10. Why Solution Works

Trie traversal guarantees:
- longest reachable path
- longest matching suffix

Aggregation guarantees:
- trie node already stores optimal answer

So query becomes:

```txt
Longest matching suffix
+
Precomputed best answer
```

which satisfies all conditions.

# 11. Final Code

```ts
// Approach: Trie and Trie aggregation (precompute answers)

function stringIndices(wordsContainer: string[], wordsQuery: string[]): number[] {

    interface TRIENODE {
        charectors?: TRIENODE[];
        ans?: number;
    }

    function getTrieNode(index: number): TRIENODE {
        const charectors = new Array(26).fill(null);
        const ans = index;

        return { charectors, ans };
    }

    const trie = getTrieNode(null);

    const visistedWords = new Set();

    // Handle no suffix matching condition
    let smallestWordlength: number = wordsContainer[0].length;
    let smallestWordIndex: number = 0;

    // Build trie + aggregate answers
    function trieAggregateSuffixWords(index: number, word: string): void {

        let node: TRIENODE = trie;

        // Reverse traversal for suffix matching
        for (let i = word.length - 1; i >= 0; i--) {

            const code = word.charCodeAt(i);
            const nodeIndex = code - 97;

            let localNode = getTrieNode(index);

            // Existing node
            if (node.charectors[nodeIndex] !== null) {
                localNode = node.charectors[nodeIndex];
            }

            // Attach node
            node.charectors[nodeIndex] = localNode;

            // Move forward
            node = localNode;

            // Aggregation logic
            const existingIndex = node.ans;

            if (existingIndex !== index) {

                const existingWord = wordsContainer[existingIndex];

                /*
                    Replace answer only if:
                    current word is shorter

                    Equal length not needed because
                    earlier index already wins.
                */
                if (word.length < existingWord.length) {
                    node.ans = index;
                }
            }
        }
    }

    // Build aggregated trie
    for (let i = 0; i < wordsContainer.length; i++) {

        const word = wordsContainer[i];

        // Global smallest word
        if (word.length < smallestWordlength) {
            smallestWordlength = word.length;
            smallestWordIndex = i;
        }

        /*
            Duplicate identical words are useless:
            earlier index already dominates.
        */
        if (!visistedWords.has(word)) {

            visistedWords.add(word);

            trieAggregateSuffixWords(i, word);
        }
    }

    // Query processing
    function checkSuffixWordQuery(word: string): number {

        let index: number = smallestWordIndex;

        let node = trie;

        for (let i = word.length - 1; i >= 0; i--) {

            const code = word.charCodeAt(i);
            const nodeIndex = code - 97;

            let localNode = null;

            // Matching suffix exists
            if (node.charectors[nodeIndex] !== null) {
                localNode = node.charectors[nodeIndex];
            }

            // No more suffix matching possible
            if (localNode === null) {
                break;
            }

            // Continue traversal
            node = localNode;

            // Aggregated answer already stored
            index = node.ans;
        }

        return index;
    }

    // Process all queries
    const ans = [];

    for (const word of wordsQuery) {
        ans.push(checkSuffixWordQuery(word));
    }

    return ans;
}
```

# 15. Key Takeaways

✅ Reverse strings for suffix problems

✅ Trie nodes can represent suffix groups

✅ Precompute answers during insertion

✅ Avoid subtree rescanning in trie problems

✅ String concatenation inside loops can secretly become `O(n²)`

✅ Trie aggregation is stronger than storing terminal words only

# 16. Common Mistakes

## ❌ BFS subtree traversal per query

Repeated rescanning causes TLE.

---

## ❌ Building suffix strings repeatedly

```ts
suffix += char
```

creates many immutable strings.

---

## ❌ Storing only terminal words

Queries need:
- best answer among descendants

not exact word only.

---

## ❌ Forgetting tie-breaking

Need:
1. shortest length
2. earliest index
