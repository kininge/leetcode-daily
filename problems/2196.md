# 🧩 Problem #2196 – Create Binary Tree From Descriptions

**Difficulty:** Medium   
**Topics:** `Tree`, `Hash Map`   
**Link:** [Create Binary Tree From Descriptions](https://leetcode.com/problems/create-binary-tree-from-descriptions);

---

## 📖 Problem Summary

Each description is given as:

```text
[parent, child, isLeft]
```

Meaning:

```text
parent
   /
child
```

if:

```text
isLeft = 1
```

or

```text
parent
      \
      child
```

if:

```text
isLeft = 0
```

Construct the complete binary tree and return its root.

---

## 💡 Intuition

The problem gives relationships in random order.

Example:

```text
20 → 15
50 → 20
20 → 17
```

Notice:

```text
20
```

appears as:

* Parent
* Child

at different times.

Therefore:

```text
We cannot immediately determine the root while constructing the tree.
```

Instead:

### First

Build all tree connections.

### Then

Find the only node that never appeared as a child.

That node must be the root.

---

## 🎯 Key Observation

In a valid tree:

```text
Every node except root
has exactly one parent.
```

Therefore:

```text
Root = Node that never appears as a child.
```

```text

                ↓

                ↓

                ↓
```

## ⚡ Approach

### Step 1 — Create/Re-use Nodes

A value may appear many times.

Example:

```text
50 → 20
20 → 15
20 → 17
```

When processing:

```text
20 → 15
```

we must re-use the already created node:

```text
20
```

instead of creating a duplicate node.

Use:

```ts
Map<number, TreeNode>
```

for node lookup.

---

### Step 2 — Build Tree Connections

For every description:

```text
[parent, child, isLeft]
```

Attach:

```text
parent.left = child
```

or

```text
parent.right = child
```

depending on:

```text
isLeft
```

---

### Step 3 — Track All Children

Whenever a node appears as:

```text
child
```

store it inside:

```ts
Set<number>
```

Example:

```text
50 → 20
50 → 80
20 → 15
```

Children:

```text
{20,80,15}
```

---

### Step 4 — Find Root

The root is the node that never appears inside:

```text
children
```

Example:

```text
Nodes:
50
20
80
15
17
19
```

Children:

```text
20
80
15
17
19
```

Missing:

```text
50
```

Therefore:

```text
Root = 50
```

## ✅ Why Solution Works

Every node except the root appears exactly once as a child.

During construction:

```text
children set
```

contains every non-root node.

Therefore:

```text
Node not in children set
```

must be the root.

Since the input guarantees a valid tree:

```text
Exactly one such node exists.
```

---

## 💻 Code

```ts
function createBinaryTree(descriptions: number[][]): TreeNode | null {
    const nodes = new Map();
    const children = new Set();

    for(const [parent, child, isLeft] of descriptions){

        // Create nodes if they don't exist
        if(!nodes.has(parent)){
            nodes.set(parent, new TreeNode(parent));
        }

        if(!nodes.has(child)){
            nodes.set(child, new TreeNode(child));
        }

        const parentNode = nodes.get(parent);
        const childNode = nodes.get(child);

        // Connect parent and child
        if(isLeft === 1){
            parentNode.left = childNode;
        }else{
            parentNode.right = childNode;
        }

        // Mark child
        children.add(child);
    }

    let root = null;

    nodes.forEach((node, value) => {
        if(!children.has(value)){
            root = node;
        }
    });

    return root;
}
```

| Space         | `O(n)`     |

Where:


## 📝 Key Takeaways

* Build nodes only once and re-use them.
* HashMap is perfect for value → TreeNode lookup.
* Root detection becomes easy by tracking all child nodes.
* A tree can be viewed as a directed graph:

```text
Parent → Child
```

* Root is simply the node with no incoming edge.

---

## ⚠️ Common Mistakes

### Creating Duplicate Nodes

Wrong:

```text
20 → 15
20 → 17
```

Creating two separate nodes for:

```text
20
```

breaks the tree structure.

Always re-use existing nodes.

---

### Forgetting to Track Children

Without tracking children:

```text
Finding root becomes difficult.
```

---

### Assuming First Parent Is Root

Example:

```text
20 → 15
50 → 20
```

Initially:

```text
20
```

looks like root.

Later:

```text
50
```

becomes the actual root.

Descriptions can arrive in any order.

---

### Confusing Tree Problem with DFS Problem

This problem is not about traversing a tree.

The real challenge is:

```text
Constructing the tree.
```

Once the tree is built, finding the root is straightforward.
