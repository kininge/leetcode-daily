# 🧩 Problem #1582 – Special Positions in a Binary Matrix

**Difficulty:** Easy  
**Topics:** `Array`, `Matrix`  
**Link:** https://leetcode.com/problems/special-positions-in-a-binary-matrix/

---

## 📝 Problem Summary

You are given an `m × n` binary matrix `mat`.

A position `(i, j)` is considered **special** if:

- `mat[i][j] == 1`
- All other elements in **row `i`** are `0`
- All other elements in **column `j`** are `0`

In other words, the `1` at `(i, j)` must be the **only `1` in its entire row and column**.

Return the **total number of such special positions**.

---

### Example

Input:
```txt
mat = [
        [1,0,0],
        [0,0,1],
        [1,0,0]
      ]
```
Output: `1`
Explanation:

- `(0,0)` → column contains another `1` → ❌ not special  
- `(1,2)` → only `1` in its row and column → ✅ special  
- `(2,0)` → column contains another `1` → ❌ not special  

Total special positions = **1**

---

### Constraints

- `1 <= m, n <= 100`
- `mat[i][j]` is either `0` or `1`

---

### Key Observation

To determine whether a cell `(i, j)` is special, we only need:

- Number of `1`s in row `i`
- Number of `1`s in column `j`

If both counts equal `1`, then `(i, j)` is a special position.

---

## 💡 Intuition

A position `(i, j)` is special only if:
```txt
mat[i][j] == 1
AND
row i contains exactly one '1'
AND
column j contains exactly one '1'
```

Instead of repeatedly scanning rows and columns for every `1`, we can precompute how many `1`s appear in each row and column.

### Key Idea

1. Traverse the matrix once and count:
   - `rows[i]` → number of `1`s in row `i`
   - `cols[j]` → number of `1`s in column `j`

2. Traverse the matrix again:
   - If `mat[i][j] == 1`
   - And `rows[i] == 1`
   - And `cols[j] == 1`
   - Then `(i, j)` is a **special position**

This avoids scanning entire rows and columns repeatedly.

---

### Why this works

The condition for a special position depends only on **counts**, not positions of other elements.

Once we know:
- how many `1`s exist in a row
- how many `1`s exist in a column

we can determine if a cell is special in **O(1)** time.

---

## 🛠 Approach 1 — Count Ones in Rows and Columns

### Idea

We perform two passes over the matrix.

**First pass**

Count how many `1`s appear in each:

- Row → `rows[i]`
- Column → `cols[j]`

This gives us the information needed to quickly verify whether a position could be special.

---

**Second pass**

Traverse the matrix again and check each cell `(i, j)`.

A cell is special if:
```txt
mat[i][j] == 1
AND
rows[i] == 1
AND
cols[j] == 1
```

If all conditions hold, increment the answer.

---

### Why it works

The definition of a special position depends only on whether the `1` is **unique in both its row and its column**.

By precomputing row and column counts, we avoid repeatedly scanning rows and columns for each cell.

This reduces the verification of a cell to a constant-time check.

---

### Code (TypeScript)

```ts
function numSpecial(mat: number[][]): number {
    const m = mat.length;
    const n = mat[0].length;

    const rows = Array(m).fill(0);
    const columns = Array(n).fill(0);

    // O(m*n) : travrase and build map the matrix
    for(let i=0; i<m; i++){
        for(let j=0; j<n; j++){
            if(mat[i][j] === 1) {
                rows[i] += 1;
                columns[j] += 1;
            }
        }
    }

    let specialCells:number = 0;
    // O(m*n) : travrase and find the special cells
    for(let i=0; i<m; i++){
        for(let j=0; j<n; j++){
            if(mat[i][j] === 1 && rows[i] === 1 && columns[j] === 1) specialCells++;
        }
    }

    return specialCells;
};
```

### Complexity

  - **Time Complexity:**
    - First pass over matrix → `O(m * n)`
    - Second pass over matrix → `O(m * n)`
    - Total → `O(m * n)`
  - **Space Complexity:**
    - Row counts array → `O(m)`
    - Column counts array → `O(n)`
    - Total → `O(m + n)`
