# 🧩 Problem #3286 – Find a Safe Walk Through a Grid

**Difficulty:** Medium      
**Topics:** `Graph`, `Breadth First Search`, `Matrix`   
**Link:** https://leetcode.com/problems/find-a-safe-walk-through-a-grid/   

---

## 💡 Intuition

At first glance, this looks like a normal grid traversal problem.

However, the goal is **not** to minimize the number of steps.

Instead, every unsafe cell (`1`) reduces our health by **1**, so we actually want to **minimize the total damage** taken while reaching the destination.

The key observation is:

- Reaching the same cell with **more remaining health** is always better.
- Reaching it again with **less or equal health** can never produce a better answer.

So instead of marking a cell as simply *visited*, we remember the **best remaining health** with which we've ever reached that cell.

Whenever we find a path that reaches the same cell with **more remaining health**, we continue exploring from there.

This idea is known as **State Relaxation**.

## ✅ My Solution

```ts
// we try breadth first search
function findSafeWalk(grid: number[][], health: number): boolean {
    const n = grid.length;
    const m = grid[0].length;

    const queue = new MyQueue();
    const remainingHealth = health-grid[0][0];
    const bestHealth = Array.from({ length: n },() => new Array(m).fill(0));
    queue.enqueue({x:0, y:0, health:remainingHealth});
    bestHealth[0][0] = remainingHealth;
    
    const row = [-1, 0, 1, 0];
    const col = [0, 1, 0, -1];

    while(queue.peek()){
        const node = queue.dequeue();
        const {x, y, health} = node as {x:number, y:number, health:number};

        // reached destination
        if(x===(n-1) && y===(m-1) && health>0) return true;

        // add adjecent cells
        if(health > 0){
            for(let i=0; i<4; i++){
                const _x = x+row[i];
                const _y = y+col[i];
                
                if(_x>=0 && _x<n && _y>=0 && _y<m){
                    const remainingHealth = health-grid[_x][_y];
                    if(bestHealth[_x][_y] < remainingHealth){
                        queue.enqueue({x:_x, y:_y, health: remainingHealth});
                        bestHealth[_x][_y]= remainingHealth;
                    }
                }
            }
        }
    }

    return false;
}

type NODE<T> = {
    next: NODE<T> | null;
    prev: NODE<T> | null;
    val: T;
}
class MyQueue<T> {
    private _size:number;
    private _head: NODE<T> | null;
    private _tail: NODE<T> | null;

    constructor(){
        this._size = 0;
        this._head = null;
        this._tail = null;
    }

    enqueue(val: T): void {
        const newNode: NODE<T> = {
            next: null,
            prev: this._tail,
            val
        };

        if (this._tail) {
            this._tail.next = newNode;
        } else {
            this._head = newNode;
        }

        this._tail = newNode;
        this._size++;
    }

    dequeue(): T|null {
        let headNodeVal: T|null = null;

        if(this._head){
            headNodeVal = this._head.val;
            this._head = this._head.next;

            if(this._head === null){
                this._tail = null;
            } else {
                this._head.prev = null;
            }

            this._size--;
        }

        return headNodeVal;
    }

    peek(): T | null {
        return this._head ? this._head.val : null;
    }

    isEmpty(): boolean{
        return this._head === null;
    }

    size(): number{
        return this._size;
    }
}
```

## 🔄 Alternative Approach

### My Approach (Accepted)

| Metric | Value |
| ------- | ----- |
| Time | O(m × n × Relaxations) |
| Space | O(m × n) |

Uses **BFS + State Relaxation**.

A cell is revisited only if we reach it with **more remaining health** than before.

---

### Optimal Approach

| Metric | Value |
| ------- | ----- |
| Time | O(V + E) |
| Space | O(V) |

Use **0-1 BFS** with a **Deque**.

Reason:

- Entering a safe cell costs **0**
- Entering an unsafe cell costs **1**

Since edge weights are only **0** and **1**, 0-1 BFS is more efficient than repeatedly relaxing states using a normal queue.

## ⚖️ Approach Comparison

| Approach | Time | Notes |
| -------- | ---- | ----- |
| DFS | Exponential | Explores too many paths |
| Plain BFS | ❌ Incorrect | Doesn't minimize damage |
| BFS + State Relaxation | Good | Accepted solution |
| 0-1 BFS | Optimal | Best solution for 0/1 edge weights |
| Dijkstra | O(V log V) | General weighted graph solution |

## 🏆 Key Takeaways

✅ Don't blindly mark graph nodes as visited.

Sometimes the **state** matters.

```text
Visited
    ❌

Best State
    ✅
```

---

For graph problems ask yourself:

```text
Is reaching the same node again useful?
```

If the new path has a **better state**, then revisiting the node is necessary.

- Bellman-Ford

- 0-1 BFS
