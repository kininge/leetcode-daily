# 🧩 Problem #3689 – Maximum Total Subarray Value I

**Difficulty:** Medium      
**Topics:** `Array`, `Greedy`      
**Link:** [Maximum Total Subarray Value I](https://leetcode.com/problems/maximum-total-subarray-value-i/description)

---

## 📖 Problem Summary

We need to choose exactly:

```text
k
```

non-empty subarrays.

The value of a subarray is:

```text
max(subarray) - min(subarray)
```

The same subarray may be chosen multiple times.

Return the maximum possible total value.

---

## 💡 Intuition

At first glance, this looks like a:

```text
Range Maximum
Range Minimum
Subarray
```

problem.

Natural thoughts are:

* Segment Tree
* Sparse Table
* Monotonic Stack

because we need:

```text
max(subarray) - min(subarray)
```

---

But there is one sentence that changes everything:

```text
The exact same subarray
can be chosen more than once.
```

This means:

```text
If we find the best subarray once,
we can simply choose it k times.
```

So the problem becomes:

```text
Find the maximum possible value
of a single subarray.
```

---

## 🎯 Key Observation

The value of any subarray is:

```text
max(subarray) - min(subarray)
```

A subarray can never have:

```text
max(subarray) > globalMaximum
```

or

```text
min(subarray) < globalMinimum
```

Therefore:

```text
max(subarray) - min(subarray)
≤
globalMaximum - globalMinimum
```

---

Can we achieve:

```text
globalMaximum - globalMinimum
```

?

Yes.

Take the subarray spanning between:

```text
globalMinimum
and
globalMaximum
```

(wherever they occur).

That subarray contains:

```text
minimum = globalMinimum
maximum = globalMaximum
```

Therefore:

```text
Best Subarray Value
=
globalMaximum - globalMinimum
```

---

## 🚀 Evolution of Thought Process

```text
Need k subarrays
        ↓
Can choose same subarray multiple times
        ↓
Only need best single subarray
        ↓
Best value = max - min
        ↓
Largest possible max = globalMax
Smallest possible min = globalMin
        ↓
Best value = globalMax - globalMin
        ↓
Answer = k × (globalMax - globalMin)
```

## ✅ Why Solution Works

For every subarray:

```text
max(subarray)
≤
globalMaximum
```

and

```text
min(subarray)
≥
globalMinimum
```

Therefore:

```text
max(subarray) - min(subarray)
≤
globalMaximum - globalMinimum
```

So no subarray can have a larger value.

Since we can choose the same subarray repeatedly:

```text
Maximum Total Value
=
k × (Best Subarray Value)
```

which becomes:

```text
k × (globalMaximum - globalMinimum)
```

---

## 💻 Code

```ts
function maxTotalValue(nums: number[], k: number): number {
    let minNum = 1_000_000_000;
    let maxNum = 0;

    for(const num of nums){
        minNum = Math.min(minNum, num);
        maxNum = Math.max(maxNum, num);
    }

    const diff = maxNum - minNum;

    return diff * k;
}
```

## 📝 Key Takeaways

* Read every sentence carefully.
* "Can choose the same subarray multiple times" is the crucial clue.
* The problem collapses into finding the best single subarray.
* Global maximum and global minimum immediately give the answer.
* Not every subarray problem requires advanced data structures.

---

## ⚠️ Common Mistakes

### Overthinking With Segment Trees

A common first thought:

```text
Need max(subarray)
Need min(subarray)
```

Therefore:

```text
Segment Tree
```

But the repeat-selection rule makes all of that unnecessary.

---

### Missing the Repetition Rule

If the same subarray could not be chosen multiple times, the problem would be completely different.

The entire solution depends on this observation.

---

### Searching All Subarrays

Trying:

```text
Enumerate subarrays
```

is unnecessary.

The answer depends only on:

```text
globalMinimum
globalMaximum
```

of the array.
