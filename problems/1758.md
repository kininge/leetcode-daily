# 🧩 Problem #1758 – Minimum Changes To Make Alternating Binary String

**Difficulty:** Easy  
**Topics:** `String`, `Greedy`    
**Link:** https://leetcode.com/problems/minimum-changes-to-make-alternating-binary-string/

---

## 📝 Problem Summary

You are given a binary string `s` consisting only of characters `'0'` and `'1'`.

In one operation, you may flip any character:
- `'0' → '1'`
- `'1' → '0'`

A string is **alternating** if no two adjacent characters are the same.

Examples of alternating strings:
```txt
010
101
010101
```
Examples that are **not** alternating:
```txt
00
0100
111
```

Your task is to determine the **minimum number of operations** required to transform `s` into an alternating binary string.

---

### Key Observation

For any binary string of length `n`, there are **only two valid alternating patterns**:
```txt
Pattern A → 010101...
Pattern B → 101010...
```

So the problem reduces to counting how many characters differ from each of these two patterns and choosing the minimum.

---

### Constraints

- `1 ≤ s.length ≤ 10⁴`
- `s[i] ∈ {'0', '1'}`

---

## 💡 Intuition

An alternating binary string means: `s[i] != s[i+1]`

For any string of length `n`, only **two alternating patterns** are possible:
```txt
Pattern A → 010101...
Pattern B → 101010...
```

Any valid alternating string must match one of these two patterns.

---

### Strategy

We compare the given string `s` with both patterns:

1️⃣ Assume the string should start with `'0'`  
```txt
Expected: 0 1 0 1 0 1 ...
```

2️⃣ Assume the string should start with `'1'`  
```txt
Expected: 1 0 1 0 1 0 ...
```

For each pattern, count how many characters in `s` differ from the expected character.

Each mismatch requires **one flip operation**.

---

### Final Decision

The minimum number of operations needed is: `min(mismatches_with_pattern_A, mismatches_with_pattern_B)`

This works because flipping characters independently allows us to transform the string into whichever alternating pattern requires fewer changes.

---

## 🛠 Approach 1 — Greedy Pattern Matching

### Idea

Since only two alternating patterns are possible: 
```txt
010101...
101010...
```

We compute how many positions differ from each pattern.

For every index `i`:

- If `i` is even:
  - Pattern A expects `'0'`
  - Pattern B expects `'1'`

- If `i` is odd:
  - Pattern A expects `'1'`
  - Pattern B expects `'0'`

Whenever the character in `s` does not match the expected character, we count one operation.

At the end, the minimum operations needed is the smaller mismatch count between the two patterns.

---

### Why it works

An alternating string must match exactly one of the two valid patterns.

Since flipping a character fixes a mismatch independently, counting mismatches directly gives the number of operations required.

---

### Code (TypeScript)

```ts
function minOperations(s: string): number {
    let oprationsForSolution1:number = 0;
    let oprationsForSolution2:number = 0;

    for(let i=0; i<s.length; i++){
        const isOdd = ((i&1) > 0); // if 0th index is set bit then odd number
        
        if(isOdd){
            if(s[i] === '1') oprationsForSolution1++;
            else oprationsForSolution2++;
        } else {
            if(s[i] === '1') oprationsForSolution2++;
            else oprationsForSolution1++;
        }

    }
    return Math.min(oprationsForSolution1, oprationsForSolution2);
};
```

### Complexity

- **Time Complexity:**
  - Single pass through the string → `O(n)`
- **Space Complexity:**
  - Only counters used → `O(1)`
