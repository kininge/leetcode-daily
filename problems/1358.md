# 🧩 Problem #1358 – Number of Substrings Containing All Three Characters

**Difficulty:** Medium   
**Topics:** `String`, `Sliding Window`, `Two Pointers`   
**Link:** https://leetcode.com/problems/number-of-substrings-containing-all-three-characters/   

---

## 💡 Intuition

We need to count all substrings that contain at least one:

```text
a
b
c
```

The important observation is:

```text
Once a substring becomes valid,

Every extension of it

is also valid.
```

Example:

```text
abcabc
```

If:

```text
abc
```

is valid,

then so are:

```text
abca

abcab

abcabc
```

Instead of checking every substring individually, we only need to find the **first valid ending**.

## 🚀 Evolution of Thought Process

### First Attempt (TLE)

```text
Choose Left
      ↓
Expand Right
      ↓
Count a, b, c
      ↓
First Valid Window
      ↓
Count Remaining
```

Time Complexity:

```text
O(n²)
```

Even though we stopped after finding the first valid window, we restarted the search for every new left index.

---

### Key Observation

Suppose:

```text
Left = i

First valid Right = j
```

Now move:

```text
Left

↓

i + 1
```

Does:

```text
Right
```

need to restart from the beginning?

No.

It can only move:

```text
→
→
→
```

Never backwards.

---

### Final Idea

```text
Expand Right
        ↓
Window becomes valid
        ↓
Count all remaining substrings
        ↓
Shrink Left
        ↓
Repeat
```

Now both pointers move only forward.

Time Complexity becomes:

```text
O(n)
```

---

## 🧠 Dry Run

Example:

```text
s = "abcabc"
```

Window:

```text
[a b c]
 L     R
```

Window is valid.

All substrings starting at `L` are:

```text
abc
abca
abcab
abcabc
```

Count:

```text
n - right + 1
```

Then shrink:

```text
remove 'a'

↓

Window invalid
```

Expand again until valid.

Repeat.

## ✅ My Solution

```ts
function numberOfSubstrings(s: string): number {
    const n = s.length;

    let count = 0;

    let a = 0;
    let b = 0;
    let c = 0;

    function updateChar(char: string, add: boolean): void {
        if(char === 'a') add ? a++ : a--;
        else if(char === 'b') add ? b++ : b--;
        else add ? c++ : c--;
    }

    let left = 0;
    let right = 0;

    while(true){

        // Expand until window contains all three characters
        while((right < n) && ((a === 0) || (b === 0) || (c === 0))){
            updateChar(s[right], true);
            right++;
        }

        if(a && b && c){
            count += (n - right + 1);
        }else{
            break;
        }

        updateChar(s[left], false);
        left++;
    }

    return count;
}
```

## ⚖️ Approach Comparison

| Approach          | Time  | Space | Notes                                 |
| ----------------- | ----- | ----- | ------------------------------------- |
| Brute Force (TLE) | O(n²) | O(1)  | Restarted search for every left index |
| Sliding Window    | O(n)  | O(1)  | Right pointer never moves backwards   |

## ⏱️ Complexity Analysis

Let:

```text
n = s.length
```

### Time Complexity

```text
O(n)
```

Both pointers move only in one direction.

```text
Left

→→→→→
```

```text
Right

→→→→→
```

Each pointer visits every character at most once.

---

### Space Complexity

```text
O(1)
```

Only three counters are maintained.

## 🏆 Key Takeaways

✅ The biggest observation was:

```text
First Valid Right
```

does **not** need to restart for the next left index.

---

✅ Whenever pointers move only:

```text
→
```

instead of:

```text
← →
```

there is a strong chance the problem can be solved using:

```text
Sliding Window
```

---

✅ Once a window satisfies:

```text
Contains a, b and c
```

every larger window ending later is also valid.

That's why:

```text
count += (n - right + 1)
```

works.

## 🚀 Interview Insight

The real breakthrough wasn't the code—it was the observation:

```text
Brute Force
      ↓
First Valid Ending
      ↓
Right Pointer Never Moves Back
      ↓
Sliding Window
      ↓
O(n)
```

This is a common optimization pattern for substring problems where the window property is **monotonic** (once valid, extending it keeps it valid).
