# 🧩 Problem #1340 – Jump Game V

**Difficulty:** Hard    
**Topics:** `Array`, `Dynamic Programming`, `Depth First Search`, `Graph`, `Topological Sort`   
**Link:** [https://leetcode.com/problems/jump-game-v/](https://leetcode.com/problems/jump-game-v/)

---

## 📝 Problem Summary

You are given:

```txt
arr
```

and an integer:

```txt
d
```

From index `i`, you may jump:

* left up to distance `d`
* right up to distance `d`

BUT:

```txt
arr[i] > arr[j]
```

must hold for destination `j`.

Additionally:

all elements between:

```txt
i and j
```

must also be:

```txt
strictly smaller than arr[i]
```

You may start from ANY index.

Return:

```txt
maximum number of indices
```

that can be visited.

---

### Example

Input:

```txt
arr = [6,4,14,6,8,13,9,7,10,6,12]
d = 2
```

One optimal path:

```txt
10 -> 8 -> 6 -> 7
```

Answer:

```txt
4
```

---

### Constraints

* `1 <= arr.length <= 1000`
* `1 <= arr[i] <= 10^5`
* `1 <= d <= arr.length`

---

## 💡 Intuition

At first glance, this problem may resemble:

* Tree traversal
* Longest path
* Graph traversal

The most important observation is:

```txt
jumps only happen from larger value -> smaller value
```

Example:

```txt
10 -> 8
8 -> 6
```

This means:

```txt
cycles are impossible
```

because values strictly decrease.

---

## 🔥 Key Insight — This Is A DAG

Since movement is always:

```txt
higher value -> lower value
```

we can never revisit a larger value.

Therefore the graph becomes:

# DAG (Directed Acyclic Graph)

This is the critical recognition point.

---

## ❌ Why This Is NOT A Tree Problem

Initially it may feel like:

```txt
connected trees + diameter problem
```

However:

* edges are directional
* multiple parents are possible
* graph may not be connected

Example:

```txt
10 -> 8
12 -> 8
```

Node `8` has:

```txt
multiple incoming edges
```

So structure is:

```txt
not a tree
```

---

## 🧠 Actual Problem Type

This problem is:

# Longest Path In DAG

Because:

* graph is acyclic
* we need maximum reachable path length

This naturally suggests:

```txt
DFS + Memoization
```

---

## 🛠 Approach — DFS + Memoization (Top-Down DP)

### DP Definition

Define:

```txt
F(i)
```

as:

```txt
maximum indices visitable
starting from index i
```

Transition:

```txt
F(i) = 1 + max(F(children))
```

where:

```txt
children = all valid jump destinations
```

---

## Step 1 — Build Graph

For every index:

search:

* left up to distance `d`
* right up to distance `d`

Stop searching when:

```txt
arr[j] >= arr[i]
```

because jumping beyond that point becomes impossible.

---

### Why Blocking Works

Condition says:

```txt
all intermediate elements
must be smaller than arr[i]
```

So once a larger/equal value appears:

```txt
further movement blocked
```

---

## Step 2 — DFS Recurrence

For every node:

try all children and compute:

```txt
1 + DFS(child)
```

Take maximum.

Memoization prevents repeated recalculation.

---

## 🔥 Why Memoization Works

Since graph is DAG:

```txt
recursive dependencies never form cycles
```

Eventually recursion always reaches:

```txt
smaller values with no outgoing edges
```

So cached results remain valid.

---

## ✅ Why This Works

Every path strictly decreases values.

DFS explores all valid decreasing paths.

Memoization guarantees:

```txt
each node solved once
```

So the longest path from every starting position is efficiently computed.

---

## 💻 Code (TypeScript)

```ts
function maxJumps(arr: number[], d: number): number {

    const n = arr.length;

    // edge case
    if (n === 1) {
        return 1;
    }

    // generate graph
    const graph = new Map<number, number[]>();

    for (let i = 0; i < n; i++) {

        const possibleIndices: number[] = [];

        // check left
        for (
            let j = i - 1;
            j >= Math.max(i - d, 0);
            j--
        ) {

            // blocked
            if (arr[j] >= arr[i]) {
                break;
            }

            possibleIndices.push(j);
        }

        // check right
        for (
            let j = i + 1;
            j < Math.min(n, i + d + 1);
            j++
        ) {

            // blocked
            if (arr[j] >= arr[i]) {
                break;
            }

            possibleIndices.push(j);
        }

        graph.set(i, possibleIndices);
    }

    // memoization
    const dp = new Array(n).fill(null);

    let largestDepthSoFar = 0;

    for (let i = 0; i < n; i++) {

        largestDepthSoFar = Math.max(
            largestDepthSoFar,
            dfs(i)
        );
    }

    function dfs(node: number): number {

        if (dp[node] !== null) {
            return dp[node];
        }

        const children = graph.get(node)!;

        let longestPath = 1;

        for (const child of children) {

            const path = 1 + dfs(child);

            longestPath = Math.max(
                longestPath,
                path
            );
        }

        dp[node] = longestPath;

        return longestPath;
    }

    return largestDepthSoFar;
}
```

---

## 🛠 Alternative Idea — Topological Sort (Bottom-Up DP)

Since graph is DAG:

Topological Sort is also possible.

Process nodes in dependency order and compute longest paths bottom-up.

---

## Why DFS + Memoization Is Preferred

Although Topological Sort works:

* implementation becomes longer
* graph ordering adds complexity
* DFS naturally follows dependency direction

So:

```txt
Top-Down DP is cleaner here
```

---

## ⚖️ DFS Memoization vs Topological DP

| Approach            | Notes                               |
| ------------------- | ----------------------------------- |
| DFS + Memoization   | Cleaner and intuitive               |
| Topological Sort DP | Valid DAG approach but more complex |

---

## ⏱ Complexity Analysis

### Graph Construction

For every index:

* scan at most `d` left
* scan at most `d` right

Complexity:

```txt
O(n * d)
```

---

### DFS + Memoization

Each node computed once.

Each edge processed once.

Complexity:

```txt
O(n * d)
```

---

## Final Complexity

| Operation       | Complexity |
| --------------- | ---------- |
| Time            | `O(n * d)` |
| Auxiliary Space | `O(n * d)` |

---

## 📌 Key Takeaways

* Strictly increasing/decreasing movement often implies:

```txt
DAG
```

* Longest path problems on DAGs are excellent candidates for:

```txt
DFS + Memoization
```

* Graph problems are not always BFS shortest-path problems.

* Tree algorithms do not automatically apply to all acyclic structures.

* Memoization converts exponential DFS into linear DAG DP.

---

## ⚠️ Common Mistakes

### 1️⃣ Treating Problem As Tree Diameter

Graph allows:

```txt
multiple parents
```

so structure is not a tree.

---

### 2️⃣ Forgetting Blocking Rule

Once:

```txt
arr[j] >= arr[i]
```

appears:

```txt
search must stop
```

---

### 3️⃣ Using BFS For Maximum Path

This is:

```txt
longest path problem
```

not shortest path.

---

### 4️⃣ Ignoring DAG Property

Strictly decreasing movement guarantees:

```txt
no cycles
```

which enables memoized DFS safely.
