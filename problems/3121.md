# 🧩 Problem #3121 – Count the Number of Special Characters II

**Difficulty:** Medium       
**Topics:** `String`, `Array`, `Hash Map`       
**Link:** [Leetcode](https://leetcode.com/problems/count-the-number-of-special-characters-ii/)

---

# 1. Problem Summary

A letter `c` is considered **special** if:

- it appears in lowercase
- it appears in uppercase
- **every lowercase occurrence** appears before the **first uppercase occurrence**

Return the number of special characters.

---

# 2. Intuition

At first glance, this looks similar to Problem 3120:

```txt
Does lowercase and uppercase both exist?
```

But there is an additional ordering condition:

```txt
ALL lowercase occurrences
must come BEFORE
FIRST uppercase occurrence
```

This changes the problem from:

- simple presence tracking

to:

- ordering validation

---

# 3. Key Observation

Instead of simulating validity during traversal, reduce the condition mathematically.

For a character to be special:

```txt
last lowercase index < first uppercase index
```

Why?

Because:

- if the LAST lowercase character still appears before
- the FIRST uppercase character

then automatically:

```txt
all lowercase occurrences are before all uppercase occurrences
```

This completely simplifies the problem.

---

# 4. Approach

We maintain two arrays:

| Array | Meaning |
|---|---|
| `lowerCaseLastOccarance[i]` | Last occurrence index of lowercase letter |
| `upperCaseFirstOccarance[i]` | First occurrence index of uppercase letter |

---

## Traversal

For every character:

### Lowercase
Update:

```txt
last lowercase position
```

### Uppercase
Store only the first occurrence.

---

## Final Validation

For every character `a → z`:

```txt
if(lastLowercase < firstUppercase)
```

then the character is special.

---

## Example

```txt
word = "aaAbcBC"
```

---

## Lowercase Last Occurrences

| Character | Last Lowercase Index |
|---|---|
| a | 1 |
| b | 3 |
| c | 4 |

---

## Uppercase First Occurrences

| Character | First Uppercase Index |
|---|---|

## Validation

| Character | Condition | Result |
|---|---|---|
| a | `1 < 2` | ✅ |
| b | `3 < 5` | ✅ |
| c | `4 < 6` | ✅ |

```txt
3
```

# 6. Why Solution Works

For a character to satisfy the condition:

```txt
all lowercase occurrences
must appear before
the first uppercase occurrence
```

The strongest lowercase occurrence is:

```txt
last lowercase occurrence
```

The earliest uppercase occurrence is:

```txt
first uppercase occurrence
```

So checking:

```txt
lastLowercase < firstUppercase
```

is sufficient and complete.

---

# 7. Code

```ts
function numberOfSpecialChars(word: string): number {
    const n = word.length;

    // Stores last occurrence of lowercase letters
    const lowerCaseLastOccarance = new Array(26).fill(n);

    // Stores first occurrence of uppercase letters
    const upperCaseFirstOccarance = new Array(26).fill(-1);

    for (let i = 0; i < n; i++) {

        const letter = word[i];
        const asciiCode = letter.charCodeAt(0);

        const isLowerCase =
            (asciiCode >= 97) && (asciiCode <= 122);

        // Lowercase letter
        if (isLowerCase) {

            const index = asciiCode - 97;

            // Update last occurrence
            lowerCaseLastOccarance[index] = i;

        } else {

            const index = asciiCode - 65;

            // Store only first uppercase occurrence
            if (upperCaseFirstOccarance[index] === -1) {
                upperCaseFirstOccarance[index] = i;
            }
        }
    }

    let count = 0;

    for (let i = 0; i < 26; i++) {

        /*
            Character is special if:
            last lowercase occurrence
            comes before
            first uppercase occurrence
        */
        if (
            lowerCaseLastOccarance[i]
            <
            upperCaseFirstOccarance[i]
        ) {
            count++;
        }
    }

    return count;
}
```

# 8. Complexity Analysis

| Operation | Complexity |
|---|---|
| Time | `O(n)` |

## Why Space is `O(1)`?

We only store information for:
- 26 lowercase letters
- 26 uppercase letters

So memory usage is constant.

# 9. Evolution of Thought Process

## Initial Naive Thinking

A natural first attempt is:

```txt
Track validity dynamically while traversing
```

But this becomes messy because:

```txt
future lowercase characters
can invalidate previous uppercase logic
```

Example:

```txt
aAaa
```

Even though:
- uppercase appears
- lowercase also appears

the condition fails because lowercase appears later again.

---

## Better Reduction

Instead of maintaining evolving state:

```txt
Reduce problem to positional comparison
```

Key reduction:

```txt
all lowercase before uppercase
```

becomes:

```txt
last lowercase < first uppercase
```

This drastically simplifies implementation.

---

# 10. Approach Comparison

| Approach | Idea | Drawback |
|---|---|---|
| Dynamic state simulation | Continuously validate character states | Complex logic |
| Position tracking | Compare final positions only | Much cleaner |

---

# 11. Key Takeaways

✅ Ordering problems often reduce to:
- first occurrence
- last occurrence

✅ Instead of simulating state:
- derive the final mathematical condition

✅ Problems with:
```txt
"all X before Y"
```

often reduce to:

```txt
last(X) < first(Y)
```

✅ Position tracking is cleaner than dynamic invalidation.

---

# 12. Common Mistakes

## ❌ Only Checking Presence

Wrong approach:

```txt
lowercase exists && uppercase exists
```

This ignores ordering.

---

## ❌ Forgetting Future Invalidations

Example:

```txt
aAaa
```

Even though uppercase exists,
lowercase appears later again.

So `a` is NOT special.

---

## ❌ Updating Uppercase Multiple Times

We only need:

```txt
first uppercase occurrence
```

not the last one.

---

## ❌ Using Complex State Machines

Trying to dynamically maintain:
- valid
- invalid
- seen
- blocked

usually overcomplicates the solution.

Position comparison is simpler.
