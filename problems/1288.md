# 🧩 Problem #1288 - Remove Covered Intervals

**Difficulty:** Medium      
**Topics:** `Intervals`, `Sorting`, `Greedy`   
**Link:** https://leetcode.com/problems/remove-covered-intervals/description

---

## 💡 Intuition

At first glance, it feels like we need to compare every interval with every other interval.

```text
Is [a,b] covered by [c,d] ?
```

That would lead to an **O(n²)** solution.

The key observation is:

> If we sort intervals properly, we only need to compare each interval with the **largest interval seen so far**.

## 🔍 Key Observation

Suppose intervals are sorted as

```text
Left endpoint  : Ascending
Right endpoint : Descending (if left is same)
```

Example

Before sorting

```text
[1,4]
[1,8]
[1,5]
```

After sorting

```text
[1,8]
[1,5]
[1,4]
```

Now the larger interval always appears first.

Since sorting guarantees

```text
previous.left <= current.left
```

The coverage condition

```text
previous.left <= current.left
AND
current.right <= previous.right
```

reduces to checking only

```text
current.right <= previous.right
```

---

## 🧠 Greedy Strategy

Maintain

```text
rightMost
```

which represents the largest right endpoint among accepted intervals.

For every interval

```text
current.right > rightMost
```

means

```text
Not covered
Accept it
```

Otherwise

```text
current.right <= rightMost
```

means

```text
Covered
Skip it
```

## ✅ Solution

```ts
function removeCoveredIntervals(intervals: number[][]): number {
    // Sort by:
    // 1. Left ascending
    // 2. Right descending (if left is same)
    intervals.sort(([l1, r1], [l2, r2]) => {
        if (l1 !== l2) return l1 - l2;
        return r2 - r1;
    });

    let count = 1;
    let rightMost = intervals[0][1];

    for (let i = 1; i < intervals.length; i++) {
        const currentRight = intervals[i][1];

        if (currentRight > rightMost) {
            count++;
            rightMost = currentRight;
        }
    }

    return count;
}
```

---

## ⚖️ Why Sorting Works

Sorting by

```text
left ↑
right ↓
```

ensures

```text
[1,8]
comes before
[1,4]
```

instead of

```text
[1,4]
[1,8]
```

This guarantees that if two intervals have the same starting point, the larger interval is processed first.

Without sorting the right endpoint in descending order, the algorithm would incorrectly count covered intervals.

## 🏆 Key Takeaways

✅ Sorting can eliminate one comparison dimension.

Instead of checking

```text
left
and
right
```

we only check

```text
right
```

because sorting already guarantees

```text
left(previous) <= left(current)
```
